1. In `packages/db/prisma/schema.prisma`, extend the `MemberSchedule` model with coaching config fields:
   - `enableBrief Boolean @default(true)` -- toggle morning brief
   - `enableNudge Boolean @default(true)` -- toggle accountability nudges
   - `enableReflection Boolean @default(true)` -- toggle reflections (separate from reflectionIntensity which controls frequency)
   - `quietStart String?` -- HH:mm format for quiet hours start (e.g., "22:00")
   - `quietEnd String?` -- HH:mm format for quiet hours end (e.g., "07:00")
   - `dailyOutreachCount Int @default(0)` -- counter for today's bot-initiated messages
   - `lastOutreachDate DateTime?` -- date the counter was last reset

   These fields extend the existing model alongside timezone, briefTime, etc. No new model needed -- MemberSchedule already serves as per-member config.

2. In `apps/bot/src/shared/delivery.ts`, add two gate functions BEFORE the existing focus session check in `deliverDM`:

   a. **Outreach budget gate**: Add a new exported function `checkAndIncrementOutreach(db, memberId): Promise<boolean>` that:
      - Loads the member's MemberSchedule
      - If `lastOutreachDate` is not today (member's timezone), reset `dailyOutreachCount` to 0 and set `lastOutreachDate` to now
      - If `dailyOutreachCount >= 3`, return false (budget exhausted)
      - Otherwise increment `dailyOutreachCount` by 1 and return true
      - If no MemberSchedule exists, return true (allow -- defaults work without config)
      - This function is called by proactive message senders (brief, nudge, reflection) BEFORE calling deliverDM. It is NOT called inside deliverDM itself -- user-initiated replies should not consume budget.

   b. **Quiet hours gate**: Add a new exported function `isQuietHours(db, memberId): Promise<boolean>` that:
      - Loads MemberSchedule, gets quietStart and quietEnd
      - If either is null, return false (no quiet hours configured)
      - Convert current time to member's timezone
      - Check if current time falls within quietStart-quietEnd window
      - Handle overnight spans (e.g., 22:00-07:00 crosses midnight)
      - Return true if currently in quiet hours

   Keep the existing `deliverDM` function signature unchanged. The new functions are utility gates that proactive senders call before attempting delivery. This keeps deliverDM simple and avoids breaking reactive message flows (user-initiated chat should never be budget-limited or quiet-gated).

   Run `npx prisma generate` after schema changes to regenerate the client.
