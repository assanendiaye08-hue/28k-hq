1. In `apps/bot/src/modules/ai-assistant/nudge.ts`:

   a. **Stale goal detection**: Add a function `findStaleGoals(db, memberId, thresholdDays = 5): Promise<Array<{id, title, daysSinceUpdate}>>` that:
      - Queries active goals for the member
      - For each goal, checks the latest check-in that references it OR the goal's own updatedAt field
      - A goal is "stale" if it has had no progress update (no check-in mentioning it, no currentValue change) for `thresholdDays` or more
      - Returns list of stale goals with days since last activity

   b. **Graduated pullback**: Modify `sendNudge` to implement a pullback curve based on days of silence:
      - 1-3 days silent: Normal nudge behavior (current implementation)
      - 4-7 days silent: Reduce to MAX 1 nudge/day regardless of accountability level. Change tone to lighter observation, not push. Example: "Your [goal] hasn't moved in 5 days. Priorities shift -- want to adjust or recommit?"
      - 8-14 days silent: Send AT MOST 1 nudge per 3 days. Tone shifts to door-open: "Still here when you need me. Your goals are waiting."
      - 15+ days silent: Stop nudging entirely. Only respond if the member initiates. The next time they message, welcome them back without guilt.
      - Track this via `lastNudgeAt` on MemberSchedule (already exists) and `daysSinceCheckIn` calculation.

   c. **Stale goal nudge integration**: In `sendNudge`, after the existing shouldNudge check, also call `findStaleGoals`. If stale goals exist, include them in the nudge AI prompt:
      - Add stale goal names and days-since-update to the nudgeInstruction
      - Frame as observations, not accusations: "Your [goal title] hasn't seen movement in [N] days."
      - Do NOT nudge about stale goals if the graduated pullback says to stop nudging

   d. **Store nudges as conversation messages**: Ensure nudges are stored via `storeMessage` with topic 'nudge' so they appear in conversation context. The existing code stores with NUDGE_MARKER prefix -- keep that but also add topic tag.
