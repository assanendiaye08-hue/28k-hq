    **Engine (src/modules/timer/engine.ts):**
    Follow voice-tracker/tracker.ts pattern closely. Import types from buttons.ts and constants.ts.

    Export `ActiveTimer` interface (exactly as specified in 09-RESEARCH.md Pattern 1):
    - memberId, mode ('pomodoro' | 'proportional'), state ('working' | 'on_break' | 'paused')
    - workDuration, breakDuration, breakRatio
    - focus (string | null), goalId (string | null)
    - currentIntervalStart (Date), totalWorkedMs, totalBreakMs, pomodoroCount
    - dmMessageId (string | null), dmChannelId (string | null)
    - startedAt (Date)

    Module-level state:
    - `const activeTimers = new Map<string, ActiveTimer>()`
    - `const timerTimeouts = new Map<string, NodeJS.Timeout>()` (for transition scheduling)

    Export functions:
    - `createTimer(memberId, options)` -- creates ActiveTimer in map, returns it. Options: mode, workDuration, breakDuration, breakRatio, focus, goalId. Does NOT schedule timeout (caller handles that after sending DM).
    - `getActiveTimer(memberId)` -- returns ActiveTimer or undefined
    - `transitionToBreak(memberId)` -- updates state to 'on_break', accumulates totalWorkedMs from currentIntervalStart, increments pomodoroCount, sets new currentIntervalStart. Calculates breakDuration for proportional mode (totalWorkedMs since last break / ratio / 60000). Returns updated timer.
    - `transitionToWork(memberId)` -- updates state to 'working', accumulates totalBreakMs, sets new currentIntervalStart. Returns updated timer.
    - `pauseTimer(memberId)` -- if state is 'working', accumulate totalWorkedMs; if 'on_break', accumulate totalBreakMs. Set state to 'paused'. Clear any scheduled timeout. Returns updated timer or null.
    - `resumeTimer(memberId)` -- restore previous state (track pre-pause state). Set new currentIntervalStart. Returns updated timer or null. For resuming during work: re-schedule remaining work time. For break: re-schedule remaining break time.
    - `stopTimer(memberId)` -- remove from map, clear timeout. If was working, accumulate final totalWorkedMs. Returns final ActiveTimer snapshot (for persistence) or null.
    - `scheduleTransition(memberId, durationMs, onTransition)` -- clear existing timeout, set new setTimeout, store handle in timerTimeouts map. As specified in research Pattern 4.
    - `clearScheduledTransition(memberId)` -- clear and delete from timerTimeouts map.
    - `getAllActiveTimers()` -- returns Array.from(activeTimers.values()) for restart recovery.
    - `restoreTimer(memberId, timer: ActiveTimer)` -- puts timer directly into the map (for restart recovery, no validation).

    Important implementation details:
    - pauseTimer must store the pre-pause state so resumeTimer knows whether to resume into 'working' or 'on_break'. Add a `prePauseState` field to ActiveTimer for this.
    - When calculating break duration for proportional mode in transitionToBreak: use only the work time from the current work interval (not cumulative totalWorkedMs), divide by breakRatio, convert to minutes.
    - All timeout handles must be cleaned up in stopTimer.
