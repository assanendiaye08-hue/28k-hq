  <name>Task 1: Natural language timer parser</name>
  <files>
    src/modules/timer/natural-language.ts
  </files>
  <action>
    Create a two-stage parser: fast keyword pre-filter followed by AI structured output.

    **Stage 1 -- Keyword pre-filter:**
    Export `isTimerRequest(message: string): boolean`
    Returns true ONLY if the message contains explicit timer/focus keywords. This prevents the AI parser from being called on every DM (Pitfall 6 -- over-broad intent detection).

    Keyword patterns (case-insensitive):
    - /\b(start|begin|set)\b.*\b(timer|pomodoro|focus\s*session|work\s*session)\b/i
    - /\b(timer|pomodoro)\b.*\b(start|begin|set|for)\b/i
    - /\bfocus\s*(for|on)\s*\d+/i (e.g., "focus for 30 minutes")
    - /\bpomodoro\b/i (standalone "pomodoro" is always timer intent)

    Must NOT match:
    - "I'm going to start working on my project" (no timer keyword)
    - "I focused on my tasks today" (past tense, no timer keyword)
    - "How does the timer work?" (question about timer, not a start request)

    **Stage 2 -- AI structured output:**
    Export `parseTimerRequest(db, memberId, message): Promise<TimerParseResult>`

    Interface TimerParseResult:
    ```
    {
      isTimerRequest: boolean;
      mode: 'pomodoro' | 'proportional' | null;
      workMinutes: number | null;
      breakMinutes: number | null;
      focus: string | null;
    }
    ```

    Use callAI with json_schema responseFormat (exactly as shown in 09-RESEARCH.md Pattern 5):
    - memberId: use the member's ID (NOT 'system') so it counts toward their budget
    - feature: 'timer'
    - System prompt: "Parse whether this message is a request to start a timer/focus session. Extract mode (pomodoro for structured work/break cycles, proportional for free-form work where break is calculated from work time), work duration in minutes, break duration in minutes (only for pomodoro), and what they want to focus on. If the user just says 'pomodoro' or 'timer' without specifying, default to pomodoro 25/5. If they say 'focus session' or 'focus for X minutes' without specifying mode, default to pomodoro. If not a timer request, set isTimerRequest to false."
    - responseFormat with json_schema exactly as in research doc
    - Parse the JSON response, return the result
    - On any error (AI failure, parse error): return { isTimerRequest: false, ... nulls }

    Add this block:
    ```
    // Check for timer intent before regular chat
    if (isTimerRequest(message.content)) {
      // Don't start if already has an active timer
      if (getActiveTimer(memberId)) {
        await message.reply("You already have a timer running. Say 'stop timer' or use /timer stop first.");
        return;
      }

      const parsed = await parseTimerRequest(db, memberId, message.content);
      if (parsed.isTimerRequest) {
        // Start timer using parsed values
        const mode = parsed.mode ?? 'pomodoro';
        const workDuration = parsed.workMinutes ?? TIMER_DEFAULTS.defaultWorkMinutes;
        const breakDuration = parsed.breakMinutes ?? TIMER_DEFAULTS.defaultBreakMinutes;

        const timer = createTimer(memberId, {
          mode,
          workDuration,
          breakDuration,
          breakRatio: TIMER_DEFAULTS.defaultBreakRatio,
          focus: parsed.focus,
          goalId: null,
        });

        // Send DM with timer embed and buttons (same as command start)
        const dmMessage = await message.reply({
          embeds: [buildTimerEmbed(timer)],
          components: [buildWorkButtons()],
        });

        // Store DM message reference on timer
        timer.dmMessageId = dmMessage.id;
        timer.dmChannelId = dmMessage.channelId;

        // Persist ACTIVE record for restart recovery
        await createActiveTimerRecord(db, timer);

        // Schedule work->break transition
        // Import and call the orchestration function from timer/index.ts
        // OR: emit a timerStarted event and let the timer module handle scheduling
        // Preferred approach: emit timerStarted and have the timer module's event listener
        // call scheduleTransition. This keeps orchestration in the timer module.
        ctx.events.emit('timerStarted', memberId, mode);

        return; // Don't process as regular chat
      }
      // If AI says it's not actually a timer request, fall through to regular chat
    }
    ```
