**1. Create src/modules/xp/engine.ts** implementing the core `awardXP` function:

```
awardXP(db, memberId, amount, source: XPSource, description): Promise<{ newTotal, leveledUp, newRank? }>
```

- Use Prisma `$transaction` for atomicity (research Pattern 1):
  1. Create XPTransaction record with memberId, amount, source, description
  2. Update Member totalXp with `{ increment: amount }`
  3. Check for level-up by comparing old rank (totalXp - amount) vs new rank (totalXp) using RANK_PROGRESSION from shared/constants.ts
  4. Return { newTotal, leveledUp, newRank if changed }
- Import RANK_PROGRESSION from shared/constants.ts for threshold lookups
- Create helper `getRankForXP(xp: number)` that returns the highest rank whose xpThreshold <= xp
- Create helper `getNextRankInfo(xp: number)` that returns "XP to next rank" string or "Max rank" if at Legend
- Create `calculateCheckinXP(dayIndex: number, streakMultiplier: number)` that applies diminishing returns from XP_AWARDS.checkin (1st = base * multiplier, 2nd = base * secondMultiplier * multiplier, etc.)
- Create `calculateStreakMultiplier(currentStreak: number)` using STREAK_CONFIG (1.0 + growth * streak, capped at maxMultiplier)

**2. Create src/modules/xp/rank-sync.ts** for Discord role management:

```
syncRank(client, db, memberId, newRankName, oldRankName): Promise<void>
```

- Fetch the member's Discord accounts from DB
- For each account, fetch the guild member via client
- Remove old rank role, add new rank role
- Wrap in try/catch per the research "Don't Hand-Roll" section (roles can be deleted by admins, bot may lack permission)
- Use RANK_PROGRESSION to find role names
- Log warnings if role not found or permission denied, do not throw

Create `buildLevelUpEmbed(displayName, newRank, totalXp)` following the research example: brand color from rank, title "Level Up!", description with member name and new rank, fields for Total XP and Next Rank, footer "Keep grinding." Delivered to private space only -- never posted publicly.
