Check-in and goal-setting commands with AI category extraction, flexible streak tracking, and XP integration.

- Use `deepseek/deepseek-v3.2` model via OpenRouter SDK (same pattern as ai-tags.ts)
- System prompt: "Extract 1-3 activity categories from this check-in. Categories should be concise (1-3 words). Examples: coding, cold outreach, content creation, learning, client work, design. **2. Create src/modules/checkin/streak.ts** -- Flexible streak tracking per research "Streak Mechanics Design":

- `updateStreak(db, memberId, timezone: string): Promise<{ currentStreak: number, multiplier: number, streakBonus: number, isComeback: boolean }>`
  - Get member's lastCheckInAt and currentStreak from DB
  - Compute "today" and "last check-in day" in member's local timezone using `@date-fns/tz` TZDate and date-fns v4
  - If same day: no streak change (additional check-in same day)
  - If consecutive day: increment streak, increase multiplier by STREAK_CONFIG.multiplierGrowth (capped at maxMultiplier)
  - If missed days: count grace days used this Mon-Sun week. If missed <= remaining grace days, streak maintained but multiplier does not increase. If missed > grace days, reduce multiplier by STREAK_CONFIG.decayRate per excess day (floor 1.0). If 7+ consecutive days missed, reset streak to 1.
  - "Comeback bonus": if member's last check-in was 2+ days ago and they're returning, flag isComeback=true and include STREAK_CONFIG.recoverBonus as streakBonus
  - Update Member record: currentStreak, longestStreak (max of current vs longest), lastCheckInAt
  - Check for streak milestones (7, 14, 30, 60, 90 days) and award bonus XP from STREAK_CONFIG.milestones

- `calculateGraceDays(db, memberId, timezone: string): Promise<{ used: number, remaining: number }>` -- helper to check how many grace days have been used this week (Mon-Sun in member's timezone)

- Import STREAK_CONFIG from `../xp/constants.js`
- Import TZDate from `@date-fns/tz`, `startOfWeek`, `startOfDay`, `differenceInCalendarDays` from `date-fns`

**3. Create src/modules/checkin/commands.ts** -- /checkin slash command:

- Register `/checkin` command with options:
  - `activity` (String, required): "What did you do today?"
  - `effort` (Integer, optional, min 1, max 5): "How hard did you push? (1-5)"
- Handler flow:
  1. `deferReply({ flags: MessageFlags.Ephemeral })` -- response is ephemeral since check-ins are private
  2. Look up member by Discord ID via DiscordAccount -> Member
  3. If no member found, reply "Run /setup first"
  4. Get or default MemberSchedule (timezone defaults to "UTC" if no schedule exists)
  5. Count today's check-ins for this member (in member's timezone) to determine dayIndex
  6. Call `extractCategories(activity)` for AI category extraction
  7. Create CheckIn record: content (will be encrypted by extension), effortRating, categories, dayIndex (1-indexed), xpAwarded (calculated next)
  8. Call `updateStreak(db, memberId, timezone)` to update streak
  9. Calculate XP: `calculateCheckinXP(dayIndex, streakMultiplier)` from xp/engine.ts, plus any streak bonus
  10. Call `awardXP(db, memberId, xpAmount, 'CHECKIN', description)` from xp/engine.ts
  11. If streak milestone bonus, call `awardXP` again with source 'STREAK_BONUS'
  12. Emit `checkinComplete` event with memberId, checkinId, dayIndex
  13. Update CheckIn record with xpAwarded amount
  14. Build response embed (success color):
      - Title: "Check-in logged"
      - Fields: Categories (joined), XP earned (with multiplier note if > 1.0x), Streak (current count + multiplier), Effort (rating if provided)
      - If dayIndex > 1: note "Check-in #{dayIndex} today (reduced XP)"
      - If comeback: note "Welcome back! +{bonus} comeback XP"
      - If level-up: celebratory note (from awardXP return)
  15. Send embed to member via editReply (ephemeral) -- the check-in stays private

- If the member has a private channel (not just DM), ALSO send a copy of the check-in summary embed there so it appears in their personal history. Use deliverToPrivateSpace for this.

**1. Create src/modules/goals/commands.ts** -- Four slash commands:

**`/setgoal`** -- Create a new goal:
- Options:
  - `title` (String, required): "What's the goal?"
  - `deadline` (String, required): "When? (e.g., 'end of week', 'March 30', '2 weeks')"
  - `target` (Integer, optional): "Target number (e.g., 5 for '5 cold emails')"
  - `unit` (String, optional): "Unit of measurement (e.g., 'emails', 'pages')"
- Handler:
  1. deferReply ephemeral
  2. Look up member by Discord ID
  3. Count active goals for this member. If >= 5, send advisory: "You already have {N} active goals. Consider completing some before adding more." but do NOT hard-block (soft cap per user decision)
  4. Parse deadline: use date-fns `parse` with common formats, or AI extraction for natural language ("end of week" -> next Sunday in member's timezone). If parsing fails, default to 7 days from now.
  5. Determine GoalType: if target AND unit provided -> MEASURABLE, else -> FREETEXT
  6. Create Goal record: title, description (set to title for now -- can be expanded via future edit), type, targetValue, unit, status ACTIVE, deadline
  7. Reply with goal confirmation embed: title, type, deadline formatted, progress bar if measurable (0/{target} {unit})

**`/goals`** -- List active goals:
- No options required
- Handler:
  1. deferReply ephemeral
  2. Fetch all goals for this member with status ACTIVE or EXTENDED, ordered by deadline ASC
  3. Build embed with each goal as a field: title, deadline (relative: "in 3 days"), progress for measurable (bar: [===---] 3/5 emails), status badge
  4. If no active goals: "No active goals. Use /setgoal to create one."
  5. Reply ephemeral

**`/progress`** -- Update progress on a measurable goal:
- Options:
  - `goal` (String, required, autocomplete): "Which goal?" -- autocomplete from active measurable goals
  - `value` (Integer, required): "New progress value (e.g., 3 for '3 out of 5 done')"
- Handler:
  1. deferReply ephemeral
  2. Find the goal by ID (from autocomplete) or title fuzzy match
  3. Validate: must be MEASURABLE and ACTIVE/EXTENDED
  4. Update currentValue. If currentValue >= targetValue, auto-complete: set status COMPLETED, completedAt, award goal XP via awardXP(GOAL_COMPLETE with measurableComplete amount), emit goalCompleted event
  5. If not yet complete: award progressUpdate XP (10 XP per increment -- only for the delta, not the absolute value), emit goalProgressUpdated
  6. Reply with updated progress embed

**`/completegoal`** -- Mark a free-text goal as done:
- Options:
  - `goal` (String, required, autocomplete): "Which goal?" -- autocomplete from active goals
- Handler:
  1. deferReply ephemeral
  2. Find goal, validate ACTIVE/EXTENDED
  3. Update status COMPLETED, completedAt
  4. Award XP: measurableComplete (100) for MEASURABLE, freetextComplete (75) for FREETEXT
  5. Call awardXP, emit goalCompleted event
  6. Reply with celebration embed: "Goal complete! +{xp} XP"

**Autocomplete handler** for goal selection: query active goals for the member, return as choices (id as value, title as name, truncated to 100 chars).

**2. Create src/modules/goals/expiry.ts** -- Goal expiry checker:

- `checkExpiredGoals(db, client): Promise<void>` -- called periodically (will be wired to scheduler in Plan 03)

- `handleGoalExtension(db, goalId, newDeadline: Date): Promise<Goal>` -- extends a goal
  - Update deadline, set extendedAt, reset status to ACTIVE
