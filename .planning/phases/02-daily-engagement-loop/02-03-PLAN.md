**2. Create src/modules/scheduler/manager.ts** -- Per-member scheduler lifecycle following research "Pattern 2":

- `SchedulerManager` class with:
  - Private `tasks: Map<string, Map<string, ScheduledTask>>` -- outer key is memberId, inner key is task type ('brief', 'reminder-HH:mm', 'planning')
  - `scheduleBrief(memberId, cronExpr, timezone, fn)`: create/replace brief cron task for this member. Use `cron.createTask(cronExpr, fn, { timezone })` from node-cron 4.x. Start immediately.
  - `scheduleReminder(memberId, time: string, timezone, fn)`: create check-in reminder task. Time is "HH:mm" string, converted to cron expression "mm HH * * *".
  - `schedulePlanning(memberId, timezone, fn)`: schedule Sunday planning session. Cron: "0 10 * * 0" (10:00 AM every Sunday) in member's timezone.
  - `unscheduleAll(memberId)`: stop and remove all tasks for a member.
  - `rebuildAll(schedules: MemberSchedule[], briefFn, reminderFn, planningFn)`: called on bot startup. Iterates all schedules from DB, creates tasks for each member that has them configured.
  - `updateMemberSchedule(memberId, schedule: MemberSchedule, briefFn, reminderFn, planningFn)`: called when member updates their settings. Stops all existing tasks for member, recreates from new schedule.
  - Error handling: wrap all task callbacks in try/catch, log errors via winston, never let a scheduled task crash the process.
  - Log each task creation/destruction for debugging.

**3. Create src/modules/scheduler/briefs.ts** -- Hybrid AI+template morning brief generation following research "Pattern 5":

- `buildBriefTemplate(memberData: MemberBriefData): BriefTemplate` -- assembles raw data into structured template:
  - `MemberBriefData` type: { displayName, timezone, briefTone, totalXp, currentStreak, streakMultiplier, currentRank, nextRankXp, activeGoals: Goal[], todayCheckIns: number, reminderTimes: string[], lastCheckInCategories: string[] }
  - Template includes: greeting by name, streak status (X days, Y multiplier), rank + progress to next, active goals with progress, today's reminders, any milestone approaching

- `generateBrief(memberData: MemberBriefData): Promise<string>` -- hybrid AI+template:
  1. Build template from real data
  2. Try AI generation via OpenRouter:
     - Model: `deepseek/deepseek-v3.2`
     - System prompt: `You are a ${memberData.briefTone} morning brief writer for a productivity Discord server. Keep it to 3-5 sentences. Be concise, warm, and motivating. Use the data provided -- never invent facts. Do not use emojis excessively.`
     - User message: JSON.stringify(template)
     - Use structured output or simple text completion (brief is free-form text, no schema needed)
     - `stream: false`
  3. If AI fails, fall back to `formatTemplateFallback(template)` which formats the data as a clean text brief without AI personality
  4. Cache: if a brief was already generated today for this member (check by querying -- or use a simple in-memory Map with date key), return cached version. Prevents regeneration cost.

- `sendBrief(client, db, memberId): Promise<void>` -- the full flow:
  1. Fetch member with profile, schedule, goals (active), recent check-ins, XP transactions
  2. Build MemberBriefData
  3. Call generateBrief
  4. Create branded embed (amber/gold) with brief text as description, fields for quick stats (XP, Streak, Active Goals count)
  5. Deliver via deliverToPrivateSpace

- `sendCheckinReminder(client, db, memberId): Promise<void>` -- reminder delivery:
  1. Check if member already checked in today (in their timezone). If yes, skip -- no nagging.
  2. Build a brief, friendly reminder message: "Hey {name}, you said you'd check in around now. How's it going? /checkin when you're ready." -- personal assistant tone, not nagging app (per user specifics).
  3. Deliver via deliverToPrivateSpace
  4. Do NOT send if member has already checked in today

**Important notes:**
- Morning brief does NOT include community pulse / who's active / who checked in -- that is Phase 3 (per user decision)
- Brief tone options: "coach" (motivational, direct), "chill" (casual, supportive), "data-first" (stats-forward, minimal fluff)
- All timezone operations use TZDate from @date-fns/tz
- Daily AI call cap: if member has already had 5 AI calls today, use template fallback (per research pitfall #4)

**1. Create src/modules/scheduler/planning.ts** -- Sunday planning DM conversation following the same pattern as `src/modules/onboarding/setup-flow.ts` (awaitMessages):

- `runPlanningSession(client, db, memberId): Promise<void>`:
  1. Fetch member's Discord account, open DM channel
  2. Fetch last week's data: check-in count, goals completed, goals active, streak
  3. Send opening message: "Hey {name}, it's planning time. Quick check -- how did last week go? (1-5, or just tell me)"
  4. Await response (5 min timeout via awaitMessages, filter for this user)
  5. Parse response: if 1-5, record as weekly rating. If text, acknowledge.
  6. Send: "What are your goals for this week? You can set up to 5. Just tell me naturally, like 'send 10 cold emails, finish the landing page, read 2 chapters of that book'."
  7. Await response
  8. Parse goals from natural language using AI (same structured output pattern):
     - System prompt: "Extract goals from this text. For each goal, determine if it's measurable (has a number and unit) or free-text. Return as JSON array."
     - Schema: `{ goals: [{ title, type: 'measurable'|'freetext', target?: number, unit?: string }] }`
     - Fallback: treat entire response as one free-text goal
  9. Create Goal records for each extracted goal. Default deadline: end of this week (Sunday 23:59 in member's timezone).
  10. Send: "When should I remind you to check in this week? Give me times like '9am, 3pm' or 'keep the same as last week'."
  11. Await response
  12. Parse reminder times: extract HH:mm patterns from text, or "keep"/"same" means use existing schedule
  13. Update MemberSchedule with new reminderTimes
  14. Send confirmation embed: summary of goals set (bullet list), reminder times, streak continuation note, motivational closer matching their brief tone
  15. Emit `scheduleUpdated` event so the SchedulerManager rebuilds this member's tasks

- Handle timeout at any step: "No worries, we can plan later. Your existing goals and reminders are still active."
- Handle DM blocked: log warning, skip this member's planning session
