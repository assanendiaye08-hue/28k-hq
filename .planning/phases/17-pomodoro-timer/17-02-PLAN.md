    2. **JavaScript tray click handler** (`apps/desktop/src/App.tsx`):
       - Import `listen` from `@tauri-apps/api/event`, `WebviewWindow` from `@tauri-apps/api/webviewWindow`, `moveWindow, Position` from `@tauri-apps/plugin-positioner`, and `useTimerStore` to check phase.
       - In a useEffect (in the App component, runs once), call `listen('tray-icon-clicked', async () => { ... })`.
       - Inside the handler: check if timer is active (phase !== 'idle' and phase !== undefined) by reading store state directly via `useTimerStore.getState().phase`.
       - **If timer is NOT active (idle):** toggle the main window visibility (same behavior as before -- `getCurrentWindow().isVisible()` then show/hide).
       - **If timer IS active:** show the popover window.
         - Try `WebviewWindow.getByLabel('timer-popover')`. If exists and visible, hide it. If exists and hidden, show it + setFocus + moveWindow(Position.TrayCenter). If doesn't exist, create it:
         ```
         new WebviewWindow('timer-popover', {
           url: '/timer-popover',
           width: 320,
           height: 420,
           decorations: false,
           alwaysOnTop: true,
           resizable: false,
           skipTaskbar: true,
           focus: true,
         })
         ```
         - After creation, listen for `tauri://created` event and call `moveWindow(Position.TrayCenter)`.
       - Return the unlisten function for cleanup.

    3. **Popover route** (`apps/desktop/src/App.tsx`):
       - Add a new route: `<Route path="/timer-popover" element={<TimerPopover />} />`
       - This route is NOT wrapped in AuthGate/AppShell (the popover is a bare window).
       - Create a simple `TimerPopover` component inline or as a separate file. It renders:
         - Dark background (`bg-surface-0 text-text-primary p-4 h-screen`)
         - ProgressRing (smaller, size=160) with countdown
         - Phase label and focus text
         - SessionDots
         - TimerControls (pause/resume/stop)
         - The stop action should also close the popover: after calling store.stop(), call `getCurrentWindow().close()`.
       - The popover reads from the same Zustand store -- since it is a separate window, the store state is shared via the same JavaScript context IF it is the same webview process. In Tauri, each WebviewWindow gets its own JavaScript context. So the popover needs to read timer state from persistence (plugin-store) or communicate via Tauri events.
       - **Simpler approach:** Since the popover is a React route in the same Vite app, AND Tauri WebviewWindows share the same origin, the Zustand store instance is NOT shared. Instead, use the `useTimerTick` hook which reads from the store -- but the store is a different instance.
       - **Correct approach:** The popover window loads the same React app at a different route. The store is re-initialized. On mount, the popover should call `loadTimerState()` to restore the store from persistence, then run `useTimerTick` normally. Actions (pause/resume/stop) save to persistence and call the API. The main window's store picks up changes on its next tick via persistence sync.
       - Add a `syncFromPersistence()` action to the timer store that reads from `loadTimerState()` and updates the store -- call this periodically or on window focus in the main window.
       - For the popover: on mount, call loadTimerState and restore. The tick hook drives the countdown. On stop, call clearTimerState and close the popover window. The main window detects the idle state on its next sync.
       - **Cross-window communication:** After any popover action (pause/resume/stop), emit a Tauri event: `import { emit } from '@tauri-apps/api/event'` then `emit('timer-state-changed')`. In the main window's TimerPage, listen for this event and call `syncFromPersistence()`.
