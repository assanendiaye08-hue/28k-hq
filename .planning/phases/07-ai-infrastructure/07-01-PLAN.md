<task type="auto">
  <name>Task 2: Centralized AI client with budget enforcement and model routing</name>
  <files>src/shared/ai-client.ts</files>
  <action>
Create `src/shared/ai-client.ts` -- the single shared AI client that replaces all 7 independent OpenRouter instances.

**Structure:**

1. **Single OpenRouter client instance:**
   - One lazy-initialized `OpenRouter` instance using `config.OPENROUTER_API_KEY` from `../../core/config.js`
   - Private `getClient()` function, never exported

2. **Model config with DB-backed hot-swap:**
   - In-memory cache: `{ primary: string; fallback: string; updatedAt: number } | null`
   - `getModelConfig(db)` reads `ai_primary_model` and `ai_fallback_model` from BotConfig table
   - Cache TTL: 60 seconds. If cache is fresh, return cached values
   - Defaults if BotConfig keys don't exist: primary = `'x-ai/grok-4.1-fast'`, fallback = `'deepseek/deepseek-v3.2'`
   - Export `resetModelConfigCache()` for explicit cache invalidation (used by admin set-model command)

3. **Budget check:**
   - `checkBudget(db, memberId): Promise<boolean>` -- returns true if member is within budget
   - Query MemberSchedule for member's timezone (fall back to 'UTC')
   - Use `startOfDay(new TZDate(new Date(), timezone))` for timezone-aware day boundary (same pattern as chat.ts)
   - Aggregate `db.tokenUsage` for this member since `todayStart`
   - Look up `db.memberAIBudget` for custom limit, default to 500_000
   - If memberId is 'system', always return true (system calls are never budget-limited)
   - Import `TZDate` from `@date-fns/tz` and `startOfDay` from `date-fns`

4. **Main `callAI` function (exported):**
   ```
   export async function callAI(db: ExtendedPrismaClient, options: AICallOptions): Promise<AICallResult>
   ```
   - Step 1: Check budget for member. If over budget, return `{ content: null, usage: null, model: '', degraded: true }`
   - Step 2: Get model config from DB cache
   - Step 3: Try primary model via `client.chat.send()` with `{ chatGenerationParams: { model, messages: options.messages, stream: false, ...(options.responseFormat ? { responseFormat: options.responseFormat } : {}) } }`
   - Step 4: If primary fails (catch error), log warning and try fallback model with same parameters
   - Step 5: If both fail, log error and return `{ content: null, usage: null, model: '', degraded: true }`
   - Step 6: Extract content from `completion.choices[0]?.message?.content`
   - Step 7: Extract usage from `completion.usage` (null-check -- it's optional in the SDK type). If present, extract `promptTokens`, `completionTokens`, `totalTokens`
   - Step 8: If usage is available, compute `estimatedCostUsd` using `estimateCostUsd()` from ai-templates.ts, then store `db.tokenUsage.create()` with all fields. Do this in a fire-and-forget `.catch()` to never block the response
   - Step 9: Return `{ content, usage: usageObj, model: actualModelUsed, degraded: false }`

5. **Logging:** Use winston with tag `[ai-client]`, same pattern as other modules.

**Critical details per user decisions:**
- Silent degradation: when budget exceeded, return `{ degraded: true }`. Do NOT message the member about limits
- Model routing: Grok 4.1 Fast primary, DeepSeek V3.2 fallback per user decision
- Token tracking: use real `response.usage` from SDK, not `estimateTokens(text.length / 4)`
- Always null-check `completion.usage` before accessing (SDK types it as optional)
- System-level calls (memberId='system') bypass budget checks entirely
