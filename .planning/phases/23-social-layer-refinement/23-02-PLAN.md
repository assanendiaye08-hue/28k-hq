    **Refine Jarvis personality (personality.ts):**

    Add an explicit accountability delegation section to the system prompt builder. This should be appended to the existing system prompt, not replace it. Add these principles:

    ```
    ACCOUNTABILITY DELEGATION:
    - You deliver hard truths. Friends deliver support. This is by design.
    - When a member has declining activity, missed commitments, or stale goals, state the facts directly in DMs. Example: "Your landing page goal hasn't moved in 12 days. Your consistency rate dropped from 85% to 62% this month."
    - Never soften hard truths with pleasantries. Never say "I know you're busy" or "don't worry about it." Just state the data.
    - Never post accountability callouts in public channels. All hard truths are DM-only.
    - Frame observations as data, not judgments. "Your streak broke after 2 missed days" is data. "You're falling behind" is judgment.
    - After stating data, offer a choice: "Want to adjust the goal, recommit, or archive it?" Autonomy preserves motivation.
    - Reference the member's own words and commitments back to them. "You said you'd have the API done by Friday. It's Sunday." This is mirroring, not accusation.
    ```

    **Refine nudge message generation (nudge.ts):**

    1. In `sendNudge()`, enhance the nudge instruction building to reference specific commitments:
       - Before building the nudge instruction, query the member's recent commitments:
         ```typescript
         const commitments = await db.commitment.findMany({
           where: { memberId, status: 'ACTIVE' },
           orderBy: { deadline: 'asc' },
           take: 3,
         });
         ```
       - If commitments exist with passed deadlines, include them in the nudge instruction:
         "Missed commitments: {title} (was due {date}). Reference these factually -- not as accusations."
       - If commitments exist with upcoming deadlines, include:
         "Upcoming: {title} due {date}. Mention only if relevant to the nudge."

    2. Update the consistency rate reference in nudge instructions:
       - Import `calculateConsistencyRate` from the streak module (added in Plan 01)
       - Calculate the member's consistency rate and include in nudge context:
         "Consistency rate: {rate}% over 30 days." (This gives the AI concrete data to reference)

       - 1-3 days (normal): "State what's pending. Offer a choice: recommit, adjust, or take a break. One question, not a lecture."

       - 8-14 days (sparse): Already good ("Still here when you need me"). Keep as-is.
       - 15+ days: Already stops nudging. Keep as-is.
