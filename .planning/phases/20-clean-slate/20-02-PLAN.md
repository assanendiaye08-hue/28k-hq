<objective>
Remove the private channel system (all private interactions via DMs) and add focus session signaling between desktop app and bot.

Purpose: Private channels create server clutter and split the interaction model. DMs are simpler, more private, and align with the v3.0 vision where Jarvis coaches via DM conversation. Focus signaling protects deep work by having the bot hold non-critical messages during timer sessions -- the highest-leverage behavior per the research (23-minute interruption recovery cost).

Output: DM-only delivery, simplified onboarding, focus session API endpoint, desktop timer signaling, bot delivery gate.
</objective>

**4. Refactor delivery.ts to DM-only with focus gate** -- Rewrite `apps/bot/src/shared/delivery.ts`:
- Remove the `PrivateSpace` lookup and channel delivery path entirely
- The function should ALWAYS deliver via DM to the member's first linked Discord account
- Add an optional `respectFocusSession` parameter (default: true). When true, check `member.inFocusSession` before sending. If the member is in a focus session, return false (message held). The caller can pass `respectFocusSession: false` for time-critical messages (like reminders marked as urgent).
- Rename function from `deliverToPrivateSpace` to `deliverDM` for clarity
- Keep the old name as a re-export alias for backward compatibility: `export const deliverToPrivateSpace = deliverDM;`

New signature:
```typescript
export async function deliverDM(
  client: Client,
  db: ExtendedPrismaClient,
  memberId: string,
  content: DeliveryContent,
  options?: { respectFocusSession?: boolean },
): Promise<boolean>
```

Implementation:
```typescript
// Check focus session (skip if respectFocusSession is false)
if (options?.respectFocusSession !== false) {
  const member = await db.member.findUnique({ where: { id: memberId }, select: { inFocusSession: true } });
  if (member?.inFocusSession) return false;
}

// DM delivery via first linked account
const account = await db.discordAccount.findFirst({ where: { memberId } });
if (!account) return false;
const user = await client.users.fetch(account.discordId);
await user.send(content);
return true;
```
