       - Accept body: `{ mode: 'POMODORO' | 'PROPORTIONAL', workDuration?: number, breakDuration?: number, breakRatio?: number, focus?: string, goalId?: string, targetSessions?: number }`
       - Validate with Zod:
         - `mode`: enum POMODORO or PROPORTIONAL (required)
         - `workDuration`: optional number, min `TIMER_DEFAULTS.minWorkMinutes`, max `TIMER_DEFAULTS.maxWorkMinutes`, default `TIMER_DEFAULTS.defaultWorkMinutes`
         - `breakDuration`: optional number, min `TIMER_DEFAULTS.minBreakMinutes`, max `TIMER_DEFAULTS.maxBreakMinutes`, default `TIMER_DEFAULTS.defaultBreakMinutes`
         - `breakRatio`: optional number, default `TIMER_DEFAULTS.defaultBreakRatio`
         - `focus`: optional string (max 200 chars)
         - `goalId`: optional string
         - `targetSessions`: optional number (1-12, null for unlimited)
       - CRITICAL: Check for existing active timer: `fastify.db.timerSession.findFirst({ where: { memberId: request.memberId, status: 'ACTIVE' } })`
         - If exists, return 409: `{ error: 'Timer already active', activeSession: existing }`
       - If goalId provided, verify the goal belongs to this member and is ACTIVE
       - Create TimerSession: `fastify.db.timerSession.create({ data: { memberId, mode, status: 'ACTIVE', focus, goalId, workDuration, breakDuration, breakRatio, source: 'DESKTOP', timerState: 'working', lastStateChangeAt: new Date() } })`
       - Link to current active season if one exists: `fastify.db.season.findFirst({ where: { active: true } })`
       - Return 201 with the created session

    2. `PATCH /:id` (update timer):
       - Accept body: `{ action: 'pause' | 'resume' | 'stop', focus?: string, remainingMs?: number, totalWorkedMs?: number, totalBreakMs?: number, pomodoroCount?: number, timerState?: string }`
       - Validate with Zod: `action` is required enum
       - Find session: `fastify.db.timerSession.findUnique({ where: { id: params.id } })`
       - Verify session belongs to `request.memberId` (return 403 if not)
       - Verify session is ACTIVE (return 400 if already completed/cancelled)

       On `action: 'pause'`:
       - Update: `{ timerState: 'paused', prePauseState: body.timerState || 'working', remainingMs: body.remainingMs, lastStateChangeAt: new Date(), totalWorkedMs, totalBreakMs, pomodoroCount }`

       On `action: 'resume'`:
       - Update: `{ timerState: session.prePauseState || 'working', prePauseState: null, remainingMs: null, lastStateChangeAt: new Date() }`

       On `action: 'stop'`:
       - Update: `{ status: 'COMPLETED', timerState: null, endedAt: new Date(), totalWorkedMs: body.totalWorkedMs ?? session.totalWorkedMs, totalBreakMs: body.totalBreakMs ?? session.totalBreakMs, pomodoroCount: body.pomodoroCount ?? session.pomodoroCount, lastStateChangeAt: new Date() }`
       - Calculate XP: `Math.floor(totalWorkedMs / (5 * 60 * 1000)) * XP_AWARDS.timer.xpPer5Minutes` (1 XP per 5 min of work), minimum `TIMER_DEFAULTS.minWorkMinutes * 60 * 1000` worked to earn any XP
       - Cap at `XP_AWARDS.timer.dailyCap` by checking today's existing timer XP: sum `xpAwarded` from today's COMPLETED sessions for this member
       - If XP > 0: call `awardXP(fastify.db, request.memberId, xpAmount, 'TIMER_SESSION', 'Desktop timer session: {focus}')` from @28k/shared
       - Update session's `xpAwarded` field
       - Return 200 with updated session + `{ xpAwarded, leveledUp, newRank }` from awardXP result

       Also accept optional `focus` update on any action (for "change focus" feature).

    3. `GET /active` (get active timer):
       - Find: `fastify.db.timerSession.findFirst({ where: { memberId: request.memberId, status: 'ACTIVE' } })`
       - Return 200 with session or `{ active: null }` if none

    4. `GET /history` (recent sessions):
       - Accept query: `?limit=10&offset=0`
       - Find: `fastify.db.timerSession.findMany({ where: { memberId: request.memberId, status: 'COMPLETED' }, orderBy: { endedAt: 'desc' }, take: limit, skip: offset })`
       - Return 200 with sessions array and count

    **Goals routes (apps/api/src/routes/goals.ts):**
    Export an async Fastify plugin with `preHandler: [authenticate]` on all routes.

    1. `GET /` (list goal hierarchy -- registered at `/goals` prefix):
       - Accept query: `?timeframe=WEEKLY|MONTHLY|QUARTERLY|YEARLY&status=ACTIVE`
       - Default: all ACTIVE goals
       - Query: `fastify.db.goal.findMany({ where: { memberId: request.memberId, status: status || undefined, timeframe: timeframe || undefined, parentId: null }, include: goalTreeInclude, orderBy: { deadline: 'asc' } })`
       - Use the `goalTreeInclude` pattern from the bot's hierarchy module (nested children 4 levels deep):
         ```
         const goalTreeInclude = { children: { include: { children: { include: { children: true } } } } };
         ```
       - For top-level goals only (parentId: null) -- children load through the include
       - Return 200 with goals array

    2. `POST /` (create goal):
       - Accept body: `{ title: string, description?: string, type: 'MEASURABLE' | 'FREETEXT', targetValue?: number, unit?: string, deadline: string (ISO date), timeframe?: 'YEARLY' | 'QUARTERLY' | 'MONTHLY' | 'WEEKLY', parentId?: string }`
       - Validate with Zod:
         - `title`: required string, min 1 char, max 200 chars
         - `type`: required enum
         - `targetValue`: required if type is MEASURABLE
         - `deadline`: required ISO date string
         - `timeframe`: optional enum
       - If parentId provided:
         - Verify parent exists and belongs to this member
         - Calculate depth: `parent.depth + 1`
         - Validate depth <= 3 (MAX_GOAL_DEPTH from bot hierarchy module)
         - If depth exceeds max, return 400: `{ error: 'Maximum goal depth exceeded' }`
       - Create goal: `fastify.db.goal.create({ data: { memberId, title, description, type, targetValue, unit, deadline: new Date(deadline), timeframe, parentId, depth: parentId ? parent.depth + 1 : 0 } })`
       - Return 201 with created goal

    3. `PATCH /:id` (update progress):
       - Accept body: `{ currentValue?: number, status?: 'COMPLETED' }`
       - Validate with Zod
       - Find goal: verify belongs to request.memberId and is ACTIVE
       - If `status === 'COMPLETED'`:
         - Determine XP: `type === 'MEASURABLE' ? XP_AWARDS.goal.measurableComplete : XP_AWARDS.goal.freetextComplete`
         - Call `awardXP(fastify.db, request.memberId, xpAmount, 'GOAL_COMPLETE', 'Completed goal: {title}')`
         - Update: `{ currentValue: targetValue || 100, status: 'COMPLETED', completedAt: new Date(), xpAwarded: xpAmount }`
       - If `currentValue` provided (progress update, not completion):
         - Update `currentValue` on the goal
         - Award progress XP: `awardXP(fastify.db, request.memberId, XP_AWARDS.goal.progressUpdate, 'GOAL_COMPLETE', 'Progress on: {title}')`
       - Return 200 with updated goal + xp result if awarded

    4. `GET /:id` (single goal with children):
       - Find: `fastify.db.goal.findUnique({ where: { id: params.id }, include: goalTreeInclude })`
       - Verify belongs to request.memberId
