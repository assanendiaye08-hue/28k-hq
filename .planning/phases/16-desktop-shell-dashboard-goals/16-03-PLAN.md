    Build the goals store, page, and tree components.

    **src/stores/goals-store.ts**: Zustand store for goals:
    - Interface Goal matching API response: id, title, description, type (MEASURABLE | FREETEXT), status (ACTIVE | COMPLETED | MISSED | EXTENDED), currentValue, targetValue, unit, timeframe (YEARLY | QUARTERLY | MONTHLY | WEEKLY), deadline, depth, children (Goal[]), createdAt
    - State: goals (Goal[]), isLoading, error (string | null), expandedIds (Set<string>)
    - Actions:
      - fetchGoals(timeframe?: string): call apiFetch<Goal[]>('/goals' + optional ?timeframe=X), set goals/error/loading
      - toggleExpanded(goalId: string): add or remove from expandedIds set
      - isExpanded(goalId: string): check if in expandedIds
    - Initialize expandedIds with the first 2 levels auto-expanded (depth 0 and 1) after fetch completes. Walk the tree after fetching to collect IDs at depth 0 and 1 and add to expandedIds.

    **src/components/goals/GoalNode.tsx**: Single goal node in the tree:
    - Props: goal (Goal), depth (number)
    - Use useGoalsStore for expandedIds/toggleExpanded
    - Layout: flex row with indentation based on depth (ml-{depth * 5} using style={{ marginLeft: `${depth * 20}px` }})
    - Left side:
      - Expand/collapse chevron (rotated right > when collapsed, down v when expanded) -- only if goal has children. Clicking toggles via store. Use a small SVG chevron icon.
      - If no children: small spacer to maintain alignment
    - Center:
      - Timeframe badge: small pill with "Yearly", "Quarterly", "Monthly", "Weekly" text. Color-coded: YEARLY = purple bg, QUARTERLY = blue bg, MONTHLY = teal bg, WEEKLY = brand gold bg. Use bg-opacity for subtlety (bg-brand/20 text-brand etc.)
      - Goal title text in text-text-primary
      - Status indicator: if COMPLETED show green checkmark, if MISSED show red X, if EXTENDED show amber clock
    - Right side:
      - For MEASURABLE goals: show "{currentValue}/{targetValue} {unit}" text in text-secondary, plus a ProgressBar (width ~120px) showing percentage (currentValue/targetValue * 100)
      - For FREETEXT goals: show a circle (empty if ACTIVE, filled green checkmark if COMPLETED)
    - Hover state: bg-surface-2 rounded transition
    - Render children recursively: if expanded and has children, map over goal.children rendering GoalNode with depth+1
    - Style the node row with py-1.5 for comfortable spacing

    **src/components/goals/GoalTree.tsx**: Tree container:
    - Props: goals (Goal[]) -- top-level goals
    - Render each top-level goal as GoalNode with depth=0
    - Group by timeframe if mixed: render yearly goals first, then quarterly, monthly, weekly. Use a section header between groups if multiple timeframes exist: "Yearly Goals", "Quarterly Goals", etc. in text-text-secondary uppercase text-xs tracking-wide mb-2 mt-4.
    - If no goals: empty state with "No goals yet" message and suggestion "Create goals in Discord using /goal or ask Jarvis" in text-tertiary

    **src/pages/GoalsPage.tsx**: Goals page:
    - Page title "Goals" in text-2xl font-bold text-text-primary mb-4
    - Timeframe filter: row of pill buttons (All, Yearly, Quarterly, Monthly, Weekly). Active filter highlighted with bg-brand text-surface-base, inactive with bg-surface-2 text-text-secondary. Default to "All".
    - On mount and on filter change: call goalsStore.fetchGoals(selectedTimeframe or undefined)
    - While loading: show LoadingSpinner
    - On error: show error with retry button
    - Render GoalTree with fetched goals
    - The tree sits inside a Card component for visual containment
