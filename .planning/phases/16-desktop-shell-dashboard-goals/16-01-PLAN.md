    Build the authentication layer: OAuth flow, API client, auth state, and login UI.

    **src/api/client.ts**: API fetch wrapper following 16-RESEARCH.md Pattern 4 exactly:
    - `API_BASE` from `import.meta.env.VITE_API_URL` or 'http://localhost:3001'
    - Module-level `accessToken` variable with getter/setter
    - `refreshAccessToken()`: load auth.json store, get refreshToken, POST to /auth/refresh, store rotated refresh token, update accessToken in memory, return new token or null
    - `apiFetch<T>(path, options)`: attach Bearer token, make request, on 401 auto-refresh and retry once, throw on non-ok response, return parsed JSON
    - Export: setAccessToken, getAccessToken, apiFetch

    **src/api/auth.ts**: Discord OAuth flow following 16-RESEARCH.md Pattern 3:
    - `generateCodeVerifier()`: 32 random bytes as hex via crypto.getRandomValues
    - `generateCodeChallenge(verifier)`: SHA-256 via crypto.subtle.digest, base64url encode
    - `loginWithDiscord()`:
      1. Start localhost server via `start({ ports: [28457, 28458, 28459] })` from @fabianlars/tauri-plugin-oauth
      2. Generate PKCE code verifier + challenge
      3. Build Discord OAuth URL with client_id from VITE_DISCORD_CLIENT_ID, redirect_uri as localhost, scope "identify", code_challenge S256
      4. Open in browser via @tauri-apps/plugin-opener
      5. The start() function returns a Promise that resolves with the callback URL when redirect is received -- use that to extract the authorization code
      6. Cancel the localhost server
      7. POST code + codeVerifier + redirectUri to API /auth/discord
      8. Store refresh token in @tauri-apps/plugin-store (auth.json)
      9. Return { accessToken, refreshToken, member }
    - `tryRestoreSession()`: Check auth.json for stored refreshToken, if found call /auth/refresh to get fresh accessToken, return member data or null
    - `logoutSession()`: POST /auth/logout, clear auth.json store, clear in-memory accessToken

    **src/stores/auth-store.ts**: Zustand store following 16-RESEARCH.md Pattern 5:
    - State: user (null | member object), isAuthenticated, isLoading (true initially)
    - Actions: login(user, accessToken), logout(), setLoading(bool)
    - login calls setAccessToken from client.ts
    - logout calls setAccessToken(null)

    **src/pages/LoginPage.tsx**: Full-screen centered login page with dark theme:
    - Gold ouroboros placeholder (a gold circle or SVG) centered
    - "28K HQ" title text in brand gold
    - "Login with Discord" button styled with gold background, dark text, hover state
    - On click: call loginWithDiscord(), on success call authStore.login(), navigate to /
    - Show loading spinner during OAuth flow
    - Show error message if login fails (styled red text)

    **src/App.tsx (update)**: Wire up the full auth flow:
    - On mount (useEffect), call tryRestoreSession() to check for stored refresh token
    - If session restored, call authStore.login() with returned data
    - If not, set isLoading false (show login page)
    - BrowserRouter with Routes: /login -> LoginPage (redirect to / if authenticated), / -> placeholder content (redirect to /login if not authenticated)
    - Keep the onCloseRequested hide behavior from Task 1
    - Show LoadingSpinner while isLoading is true
