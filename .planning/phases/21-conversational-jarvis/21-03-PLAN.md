1. **brainstorm.ts** -- Create the brainstorming session manager:
   - Define `BrainstormPhase` type: `'diverge' | 'cluster' | 'evaluate'`
   - Define `BrainstormSession` interface: `{ topic: string; phase: BrainstormPhase; ideas: string[]; clusters: string[][]; startedAt: number }`
   - Export `BrainstormManager` class:
     ```
     class BrainstormManager {
       private sessions = new Map<string, BrainstormSession>(); // keyed by memberId
       private SESSION_TTL = 30 * 60 * 1000; // 30 minutes

       hasActiveSession(memberId: string): boolean
       startSession(memberId: string, topic: string): string  // returns opening message
       async handleMessage(db, memberId: string, message: string): Promise<string>  // returns response
       endSession(memberId: string): void
     }
     ```
   - `startSession`: Creates session in diverge phase. Returns an opening message:
     ```
     "Brainstorm: ${topic}\n\nPhase 1: DIVERGE -- throw out every idea, no filtering.\n\nGo. What comes to mind?"
     ```
   - `handleMessage`: Routes based on current phase:
     - **Diverge phase**: Accumulate ideas from user messages. When user says "ok", "next", "what do we have", "cluster", or "group" -- transition to cluster phase. Call the LLM to cluster the accumulated ideas into groups. Return the clusters.
     - **Cluster phase**: Present clustered ideas. When user says "evaluate", "which", "best", "pick", "rank" -- transition to evaluate phase. Call the LLM to evaluate and rank the clusters. Return the evaluation.
     - **Evaluate phase**: Present evaluation. Session ends naturally. When user says "done", "thanks", "got it" -- end session and return closing message.
     - For LLM calls within brainstorm, use `callAI` with `feature: 'chat'` and a phase-specific system prompt injection:
       - Diverge: "The user is brainstorming about '${topic}'. They are in the DIVERGE phase -- help them generate more ideas. Be generative, not critical. Build on their ideas."
       - Cluster: "Group these ideas into 2-4 clusters by theme. Label each cluster. Ideas: ${ideas.join(', ')}"
       - Evaluate: "Evaluate these clusters for the topic '${topic}'. Rank by potential impact and feasibility. Be direct about tradeoffs."
   - Add TTL cleanup: check `startedAt` against `SESSION_TTL` on access, auto-end expired sessions.

2. **index.ts** -- Wire brainstorm into the DM handler:

     // Check for active brainstorm session
     if (brainstormManager.hasActiveSession(account.memberId)) {
