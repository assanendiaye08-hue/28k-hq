    **1. Update prisma/schema.prisma:**
    Add to the existing Goal model (do NOT remove or rename any existing fields):
    - `parentId String?` -- FK to parent goal (null = standalone/top-level)
    - `timeframe GoalTimeframe?` -- timeframe level (null = legacy standalone)
    - `depth Int @default(0)` -- 0 = top-level, 1-3 = nested depth
    - `parent Goal? @relation("GoalHierarchy", fields: [parentId], references: [id], onDelete: SetNull)` -- if parent deleted, children become standalone
    - `children Goal[] @relation("GoalHierarchy")`
    - `@@index([parentId])` -- fast child lookups

    Add new enum after GoalStatus:
    ```
    enum GoalTimeframe {
      YEARLY
      QUARTERLY
      MONTHLY
      WEEKLY
    }
    ```

    IMPORTANT: Keep ALL existing fields, relations, and indexes unchanged. The named relation "GoalHierarchy" is required for Prisma self-referential disambiguation.

    **2. Create src/modules/goals/hierarchy.ts:**

    Export `MAX_GOAL_DEPTH = 3` (0-indexed, so 4 levels: 0,1,2,3).

    Export `goalTreeInclude` -- a reusable Prisma include object for loading 4 levels deep:
    ```typescript
    export const goalTreeInclude = {
      children: {
        include: {
          children: {
            include: {
              children: true,
            },
          },
        },
      },
    };
    ```

    Export `validateGoalDepth(parentDepth: number): boolean` -- returns true if a child can be added (parentDepth < MAX_GOAL_DEPTH).

    Export `getTimeframeDeadline(timeframe: string): Date` -- returns the deadline for a timeframe:
    - YEARLY: end of current year (Dec 31, 23:59:59)
    - QUARTERLY: end of current quarter
    - MONTHLY: end of current month
    - WEEKLY: end of current week (Sunday)
    Use date-fns (endOfWeek, endOfMonth, endOfQuarter, endOfYear) -- all already available.

    Export `async function recalculateParentProgress(db: ExtendedPrismaClient, parentId: string, events: IEventBus): Promise<void>`:
    - Load parent with children
    - If no parent or no children, return
    - Filter countable children: status in ['ACTIVE', 'EXTENDED', 'COMPLETED'] (exclude MISSED, CANCELLED)
    - If no countable children, return (don't auto-fail parent)
    - Calculate percentage: Math.round((completedCount / countableCount) * 100)
    - Update parent's currentValue to percentage
    - If all countable children are complete (100%):
      - Set parent status to COMPLETED, completedAt to now, xpAwarded to XP_AWARDS.goal.parentAutoComplete
      - Call awardXP with 50 XP (parentAutoComplete), source 'GOAL_COMPLETE', description "Auto-completed parent goal: {title}"
      - Emit goalCompleted event
    - If parent has a parentId itself, recurse: await recalculateParentProgress(db, parent.parentId, events)

    Import IEventBus from shared/types, ExtendedPrismaClient from db/client, awardXP from xp/engine, XP_AWARDS from xp/constants.

    **3. Update src/modules/xp/constants.ts:**
    Add to the `goal` section of XP_AWARDS:
    ```

    ```
