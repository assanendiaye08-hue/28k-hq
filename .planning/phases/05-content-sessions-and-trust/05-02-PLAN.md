       ```
       // Phase 5: Lock-In Sessions
       model LockInSession {
         id              String            @id @default(cuid())
         creatorMemberId String
         voiceChannelId  String?           // Discord voice channel ID (null if scheduled but not started)
         textChannelId   String?           // Discord text channel for session announcements (for public sessions)
         title           String            // What the session is about
         visibility      SessionVisibility
         status          SessionStatus     @default(PENDING)
         scheduledFor    DateTime?         // Null for instant sessions
         startedAt       DateTime?
         endedAt         DateTime?
         durationMinutes Int?

         participants    SessionParticipant[]

         @@index([status])
         @@index([creatorMemberId])
         @@index([scheduledFor])
       }

       model SessionParticipant {
         id        String    @id @default(cuid())
         sessionId String
         memberId  String
         role      String    @default("participant") // "creator" or "participant"
         joinedAt  DateTime  @default(now())
         leftAt    DateTime?

         session LockInSession @relation(fields: [sessionId], references: [id], onDelete: Cascade)

         @@unique([sessionId, memberId])
         @@index([memberId])
       }

       enum SessionVisibility {
         PUBLIC
         PRIVATE
       }

       enum SessionStatus {
         PENDING
         ACTIVE
         COMPLETED
         CANCELLED
       }
       ```

Note: LockInSession does NOT need a Member relation -- creatorMemberId is a plain string used for queries only, since sessions should persist even if the member deletes their data (the session record itself isn't personal data). However, SessionParticipant rows won't cascade from Member deletion since there's no relation -- this is intentional (session records are communal, not personal).

    1. Create `src/modules/sessions/manager.ts`:
       - In-memory Map<string, { sessionId: string, voiceChannelId: string, creatorMemberId: string }> for active sessions keyed by sessionId
       - Export `startInstantSession(guild, db, events, creatorMemberId, creatorDiscordId, title, visibility, inviteeDiscordIds): Promise<LockInSession>`
         a. Check MAX_ACTIVE_SESSIONS_PER_MEMBER -- reject if creator already has an active session
         b. If title not provided, infer from member's most recent active goal title. If no active goals, use member's currentFocus from profile. Fallback: "General lock-in"
         c. Create LockInSession in DB with status ACTIVE, startedAt now
         d. Create voice channel via createSessionVoiceChannel
         e. Update DB record with voiceChannelId
         f. Add creator as SessionParticipant with role "creator"
         g. Store in active sessions map
         h. DM each invitee with session invite embed (via their Discord accounts, using try/catch per invitee)
         i. If PUBLIC: post announcement embed in #sessions channel, ping @LockInSessions role
         j. Emit sessionStarted event
         k. Return the session record

       - Export `scheduleSession(db, creatorMemberId, title, visibility, scheduledFor, inviteeDiscordIds): Promise<LockInSession>`
         a. Create LockInSession in DB with status PENDING, scheduledFor set
         b. DM invitees with schedule info
         c. If PUBLIC: post scheduled announcement in #sessions
         d. Return the session record (actual start happens via cron -- see module index)

       - Export `endSession(guild, db, events, client, sessionId): Promise<{ durationMinutes: number, participantCount: number } | null>`
         a. Find session in DB, verify status is ACTIVE
         b. Calculate duration from startedAt to now
         c. Update session: status COMPLETED, endedAt now, durationMinutes
         d. Update all participants who haven't left: set leftAt to now
         e. Count participants. If 2+ participants, award sessionHost XP (10 XP) to the creator via awardXP with source SESSION_HOST
         f. Build summary embed and post it:
            - In the session's voice channel (text-in-voice if available) OR #sessions channel
            - Include: title, creator name, duration, participant names, what was worked on
         g. Wait SESSION_CLEANUP_DELAY_MS, then delete voice channel via deleteSessionVoiceChannel
         h. Remove from active sessions map
         i. Emit sessionEnded event
         j. Return duration and participant count

       - Export `inviteMidSession(guild, db, sessionId, newDiscordId): Promise<void>`
         a. Verify session is ACTIVE
         b. Add permission overwrite on voice channel for the new user
         c. Create SessionParticipant record
         d. DM the new invitee with invite embed

       - Export `cleanupOrphanedSessions(guild, db): Promise<number>`
         a. Query all sessions with status ACTIVE from DB
         b. For each, check if the voice channel still exists in the guild
         c. If channel is gone or empty: end the session (set COMPLETED, calculate duration)
         d. If channel exists with people in it: add to active sessions map (reconstruct like voice-tracker)
         e. Return count of cleaned up sessions

       - Export `handleVoiceJoin(db, sessionId, memberId): Promise<void>` -- when someone joins a session voice channel, add them as participant if not already
       - Export `handleVoiceLeave(db, sessionId, memberId): Promise<void>` -- when someone leaves, set their leftAt

       Important: Do NOT award time-based voice XP in this module. Voice tracker already tracks all channels under the "Lock In" category and awards XP automatically. Session module only awards the small `sessionHost` bonus for organizing.

       Handler implementations:
