       - Add reminders Reminder[] relation on Member model

    3. Create src/modules/reminders/constants.ts:
       - BUTTON_IDS: { ACKNOWLEDGE: 'reminder:ack', SKIP_NEXT: 'reminder:skip' } as const
       - REMINDER_DEFAULTS: { maxRepeatCount: 3, repeatIntervalMs: 5 * 60 * 1000, maxTimeoutMs: 2_147_483_647 (setTimeout 32-bit limit), sweepIntervalMs: 60 * 60 * 1000 (1 hour sweep for far-future reminders) }
       - REMINDER_PATTERNS: RegExp array for intent detection: /\bremind\s+me\b/i, /\breminder\b.*\b(to|at|on|for)\b/i, /\bset\s+(a\s+)?reminder\b/i
       - RECURRENCE_PATTERNS: Array of { pattern: RegExp, cronDay: string } for "every Monday" -> "* * 1", "every Tuesday" -> "* * 2", etc., plus "every day" -> "* * *", "every weekday" -> "* * 1-5", "every weekend" -> "* * 0,6"
       - URGENCY_KEYWORDS: /\b(urgent|urgently|important|critical)\b/i for detecting high urgency in natural language

    4. Create src/modules/reminders/parser.ts:
       - Import chrono-node (import * as chrono from 'chrono-node')
       - isReminderRequest(text: string): boolean -- checks text against REMINDER_PATTERNS, returns true if any match. Also add QUESTION_PATTERNS similar to timer (e.g., /\b(how|what|when|where|why|can|does|will|is)\b.*\breminder/i) to reject questions about reminders.
       - ParsedReminder interface: { content: string, fireAt: Date | null, isRecurring: boolean, cronExpression: string | null, urgency: 'LOW' | 'HIGH' }
       - parseReminder(text: string, timezone: string): ParsedReminder | null
         - Step 1: Detect recurrence via RECURRENCE_PATTERNS. If matched, extract cronDay and strip recurrence prefix from text for chrono parsing.
         - Step 2: Detect urgency via URGENCY_KEYWORDS. Strip urgency keyword from text.
         - Step 3: Parse time with chrono.parse(text, { instant: new Date(), timezone }, { forwardDate: true }).
         - Step 4: If no chrono result, return null (unparseable time).
         - Step 5: Extract content by removing "remind me" prefix, the chrono-parsed time expression (using result.text and result.index), the recurrence prefix, urgency keywords, and leading "to". Trim.
         - Step 6: For recurring reminders, build cron expression: extract hour and minute from chrono result, combine with cronDay -> "${minute} ${hour} ${cronDay}".
         - Step 7: Return ParsedReminder with fireAt (chrono date for one-shot, or the chrono date as next occurrence for recurring), isRecurring, cronExpression, content, urgency.
