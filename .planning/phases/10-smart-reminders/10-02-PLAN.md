    1. Create src/modules/reminders/scheduler.ts:
       - In-memory Maps:
         - pendingTimeouts: Map<string, NodeJS.Timeout> for one-shot reminders
         - pendingRepeats: Map<string, NodeJS.Timeout[]> for high-urgency repeat chain
         - activeCronTasks: Map<string, ScheduledTask> for recurring reminders (import ScheduledTask from node-cron)
       - fireReminder(reminder, delivery: DiscordReminderDelivery, db, client, options?: { isDelayed?: boolean }):
         - Build appropriate buttons based on urgency and recurrence
         - Call delivery.deliver(memberId, content, urgency, { buttons, isDelayed })
         - If urgency HIGH and not acknowledged: schedule repeat chain (3 repeats, 5 min apart via setTimeout) -- store timeouts in pendingRepeats map
         - Update DB: set status to FIRED for one-shot, store dmMessageId
         - Fire-and-forget: wrap everything in try/catch, log failures, never throw
       - scheduleOneShot(reminder, delivery, db, client):
         - Calculate delayMs = fireAt.getTime() - Date.now()
         - If delayMs <= 0: fire immediately (missed reminder, pass isDelayed: true)
         - If delayMs > REMINDER_DEFAULTS.maxTimeoutMs (24.8 days): skip direct scheduling -- the hourly sweep will pick it up when it's within range
         - Otherwise: setTimeout(() => fireReminder(reminder, ...), delayMs) -- store in pendingTimeouts
       - scheduleRecurring(reminder, delivery, db, client):
         - Get member timezone from db.memberSchedule.findUnique
         - Use cron.schedule(reminder.cronExpression, callback, { timezone })
         - Callback checks skipUntil: if skipUntil > now, clear skipUntil in DB and skip silently. Otherwise fireReminder.
         - Store ScheduledTask in activeCronTasks
       - cancelReminder(reminderId: string):
         - Clear timeout from pendingTimeouts if exists
         - Clear repeat timeouts from pendingRepeats if exists
         - Stop and destroy cron task from activeCronTasks if exists
       - acknowledgeReminder(reminderId: string, db):
         - Clear repeat timeouts from pendingRepeats
         - Update DB: set acknowledged = true
       - skipNextOccurrence(reminderId: string, db):
         - Calculate next occurrence time from cron expression
         - Set skipUntil in DB to that time (or simply set skipUntil to a time that covers the next fire -- e.g., now + 23 hours for daily, now + 6 days + 23 hours for weekly)
       - recoverReminders(db, client, delivery):
         - Query pending one-shot: WHERE status = 'PENDING' AND cronExpression IS NULL
         - For each: scheduleOneShot (if fireAt < now, it fires immediately with delayed flag)
         - Query active recurring: WHERE status = 'ACTIVE' AND cronExpression IS NOT NULL
         - For each: scheduleRecurring
         - Log recovery counts
       - startSweep(db, client, delivery):
         - setInterval every REMINDER_DEFAULTS.sweepIntervalMs (1 hour)
         - Query: WHERE status = 'PENDING' AND cronExpression IS NULL AND fireAt BETWEEN now AND now + 2 hours AND id NOT IN pendingTimeouts.keys()
         - Schedule each via scheduleOneShot
