**Memory service (src/modules/ai-assistant/memory.ts):**

Create the conversation memory service with:

- `storeMessage(db, memberId, role, content)` -- save a message to ConversationMessage table
- `getRecentMessages(db, memberId, limit = 50)` -- load last N messages in chronological order
- `getSummary(db, memberId)` -- load the conversation summary if it exists
- `assembleContext(db, memberId)` -- full context assembly:
  1. Load member with profile, active goals, schedule, last 5 check-ins, last 3 voice sessions (include duration)
  2. Load conversation summary
  3. Load recent messages (last 50)
  4. Estimate token count using `Math.ceil(text.length / 4)` approximation
  5. If total tokens > 100,000 (70% of DeepSeek 164K), trim oldest recent messages first
  6. If still over budget after trimming to 10 messages, trigger `compressSummary`
  7. Return `{ systemPrompt: string, summary: string | null, recentMessages: Array, memberContext: string }`
- `compressSummary(db, memberId)` -- call DeepSeek to summarize the oldest messages into a compressed summary:
  1. Load all messages NOT in the recent 50
  2. If there's an existing summary, prepend it
  3. Call OpenRouter with a system prompt: "Compress this conversation history into a concise summary preserving key facts, commitments, preferences, and ongoing topics. Keep it under 2000 words."
  4. Upsert the ConversationSummary record with the result and update messageCount
  5. Delete the messages that were summarized (they're now in the summary)
- `exportHistory(db, memberId)` -- export all messages + summary as JSON array of `{ role, content, timestamp }`
- `wipeHistory(db, memberId)` -- delete all ConversationMessage and ConversationSummary for the member

Use the existing `getOpenRouterClient()` lazy singleton pattern from briefs.ts (or import from a shared location -- if ai-tags.ts also has one, create a shared `src/shared/openrouter.ts` that both can use, or just create a local one in this module).

Token estimation: `function estimateTokens(text: string): number { return Math.ceil(text.length / 4); }` -- simple approximation per research recommendation.

Constants:
- `RECENT_MESSAGE_CAP = 50`
- `CONTEXT_BUDGET = 100_000` (tokens, ~70% of 164K)
- `OUTPUT_RESERVE = 4_000` (tokens reserved for model output)
- `SYSTEM_PROMPT_RESERVE = 20_000` (tokens for system prompt + member context)

Use the existing OpenRouter SDK pattern: `client.chat.send({ chatGenerationParams: { model, messages, stream: false } })`.

  4. Calls OpenRouter with `deepseek/deepseek-v3.2`, `stream: false`
