**Nudge logic (src/modules/ai-assistant/nudge.ts):**

Create the accountability nudge system:

1. Export `ACCOUNTABILITY_LEVELS` constant:
   ```
   light:  { maxNudgesPerDay: 1, triggerAfterMissedDays: 2, silenceThresholdDays: 7, tone: 'gentle reminder' }
   medium: { maxNudgesPerDay: 2, triggerAfterMissedDays: 1, silenceThresholdDays: 5, tone: 'direct check-in' }
   heavy:  { maxNudgesPerDay: 3, triggerAfterMissedDays: 0, silenceThresholdDays: 3, tone: 'accountability partner' }
   ```

2. Export `shouldNudge(db, memberId, accountabilityLevel)`:
   - Check if member has checked in today (their timezone) -- if yes, return false
   - Check `lastNudgeAt` on MemberSchedule -- if already nudged today, return false
   - Count today's nudges (from ConversationMessage where role='assistant' and content contains nudge marker, createdAt >= today) -- if >= maxNudgesPerDay, return false
   - Calculate days since last check-in -- if less than triggerAfterMissedDays, return false
   - Return true

3. Export `sendNudge(client, db, memberId)`:
   - Load member with schedule, profile, goals, last check-in
   - Determine accountability level from schedule (default 'medium')
   - Call `shouldNudge` -- return early if false
   - Check for extended silence: if days since last check-in >= silenceThresholdDays, switch to "genuine check-in" mode
   - Build AI nudge message via OpenRouter:
     - For normal nudge: use Ace personality with tone adjusted per level. Include what they're missing (streak at risk, goal deadline approaching, etc.)
     - For extended silence: "Hey [name], it's been [X] days. No pressure, but I want to check in -- are you still locked in on [their goals], or do you want to adjust? You can always use `/accountability light` or just tell me to back off."
   - Deliver via `deliverToPrivateSpace` (DM only -- per user decision)
   - Update `lastNudgeAt` on MemberSchedule to prevent duplicate nudges
   - Store the nudge as a conversation message (role: 'assistant') so Ace has context of what was sent
   - Log the nudge delivery

4. Multi-account handling: When delivering nudge, use `deliverToPrivateSpace` which already picks the first linked account. For members with multiple accounts, this ensures one nudge only (not double-send per user requirement).
