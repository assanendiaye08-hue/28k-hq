   - id: String @id @default(cuid())
   - memberId: String (relation to Member, onDelete: Cascade)
   - name: String (the person they admire -- e.g., "Steve Jobs", "Kobe Bryant")
   - context: String? (optional -- why they admire this person)
   - createdAt: DateTime @default(now())
   - @@unique([memberId, name]) to prevent duplicates
   - @@index([memberId])
   - Add `inspirations Inspiration[]` relation on the Member model

2. Run `npx prisma db push` to apply the schema change.

3. Create src/modules/inspiration/commands.ts:
   - Export `buildInspirationCommand()` that returns a SlashCommandBuilder with:
     - Name: "inspiration"
     - Description: "Manage the people who inspire you"
     - Subcommands:
       - `add`: required `name` string option ("Name of the person"), optional `context` string option ("Why they inspire you")
       - `remove`: required `name` string option with autocomplete
       - `list`: no options
   - Export `handleInspirationCommand(interaction, db)`:
     - `add` subcommand:
       - Look up the member via discordId -> memberId
       - Count existing inspirations for this member
       - If count >= 3, reply ephemeral: "You can have up to 3 inspirations. Remove one first with /inspiration remove."
       - Otherwise, upsert (using the unique constraint) with the name and optional context
       - Reply ephemeral with success embed: "{name} added as an inspiration." plus context if provided
     - `remove` subcommand:
       - Delete where memberId + name matches (case-insensitive: use name.trim())
       - If nothing deleted (count 0), reply ephemeral: "No inspiration found with that name."
       - Otherwise reply ephemeral: "Removed {name} from your inspirations."
     - `list` subcommand:
       - Find all inspirations for the member
       - If empty, reply ephemeral: "No inspirations set yet. Use /inspiration add to add someone who inspires you."
       - Otherwise build an embed listing each inspiration with name and context (if any), using the brand primary color
   - For autocomplete on `remove`: return the member's current inspiration names

4. Create src/modules/inspiration/index.ts following the standard Module pattern:
   - Export name = 'inspiration'
   - register(ctx): register the 'inspiration' command and wire autocomplete handler

5. Update src/deploy-commands.ts:
   - Import buildInspirationCommand from the inspiration module
   - Add it to the commands array under a "Phase 8 Commands" section comment

Note: Use ephemeral replies for all /inspiration interactions (personal data, not for public display). Follow the existing pattern of deferReply({ flags: MessageFlags.Ephemeral }) then editReply for operations that hit the DB.
