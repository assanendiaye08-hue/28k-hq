    Create prisma/schema.prisma with ALL Phase 1 models (use the exact schema from the research doc as the baseline):

    Models to create:
    1. Member -- core identity (id: cuid, displayName, encryptionSalt: random 16-byte hex, recoveryKeyHash, createdAt, updatedAt)
    2. DiscordAccount -- linked Discord accounts (discordId as @id, memberId FK, linkedAt). Index on memberId.
    3. MemberProfile -- profile data (id: cuid, memberId: unique, rawAnswers: encrypted String, interests: String[], currentFocus, goals: String[], learningAreas: String[], workStyle, publicFields: String[], createdAt, updatedAt)
    4. PrivateSpace -- private space preference (id: cuid, memberId: unique, type: SpaceType enum DM|CHANNEL, channelId: optional)
    5. LinkCode -- temporary account link codes (id: cuid, code: unique, requesterId, expiresAt, used: default false). Index on [code, expiresAt].
    6. InterestTag -- AI-managed interest tags (id: cuid, name: unique, roleId: optional, memberCount: default 0, createdAt, lastUsedAt)

    Generator: prisma-client-js. Datasource: postgresql with env("DATABASE_URL").

    Create src/shared/crypto.ts:
    - ALGORITHM = 'aes-256-gcm', IV_LENGTH = 12, AUTH_TAG_LENGTH = 16
    - deriveMemberKey(masterKey: Buffer, memberId: string): Buffer -- uses crypto.hkdfSync('sha256', masterKey, memberId, 'discord-hustler-member-key', 32)
    - encrypt(plaintext: string, memberKey: Buffer): string -- AES-256-GCM, returns base64 packed (iv + authTag + ciphertext)
    - decrypt(packed: string, memberKey: Buffer): string -- reverses the above
    - generateRecoveryKey(memberKey: Buffer): string -- returns "DHKEY-" + memberKey.toString('base64url')
    - generateEncryptionSalt(): string -- crypto.randomBytes(16).toString('hex')
    - All functions use Node.js built-in crypto module (no external deps)

    Create src/db/encryption.ts:
    - Define ENCRYPTED_FIELDS map: { MemberProfile: ['rawAnswers'] } (only rawAnswers is encrypted -- structured tags are intentionally cleartext for queries)
    - Export withEncryption(prisma: PrismaClient) function that returns prisma.$extends({...})
    - In the query.$allOperations hook:
      - On write operations (create, update, upsert): find memberId from args.data or args.where, derive member key, encrypt each field in ENCRYPTED_FIELDS[model]
      - After query execution: decrypt fields in the result
      - Get MASTER_ENCRYPTION_KEY from config (import from core/config.ts)
      - Handle nested writes and batch operations gracefully
    - IMPORTANT: The master key MUST come from environment variables, never hardcoded

    Replace the placeholder src/db/client.ts (from Plan 01):
    - Create PrismaClient singleton
    - Apply withEncryption extension
    - Export the extended client as `db`
    - Add connection handling (connect on first use, disconnect helper for cleanup)

    After creating schema, run: npx prisma generate (to generate the client types)
    NOTE: Do NOT run prisma migrate yet -- that requires a running PostgreSQL. Just generate the client.
