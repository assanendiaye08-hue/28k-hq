  <action>
    Create src/core/config.ts:
    - Use zod to validate all environment variables at startup
    - Load from .env via dotenv (development) or real env vars (production)
    - Export a typed `config` object. Crash immediately with descriptive error if required vars are missing.
    - Required: BOT_TOKEN, DISCORD_CLIENT_ID, DISCORD_GUILD_ID, DATABASE_URL, OPENROUTER_API_KEY, MASTER_ENCRYPTION_KEY

    Create src/core/client.ts:
    - Export a `createClient()` function that returns a configured Discord.js Client
    - Intents: Guilds, GuildMembers (privileged), GuildMessages, DirectMessages, MessageContent (privileged)
    - Partials: Channel, Message (both required for DM events)
    - Do NOT login inside createClient -- the caller (index.ts) handles that

    Create src/core/module-loader.ts:
    - Export a `loadModules(ctx: ModuleContext)` function
    - Dynamically import all index.ts files from src/modules/*/
    - Each module must export a default Module object (or named `module`)
    - Call module.register(ctx) for each discovered module
    - Log which modules were loaded using winston
    - Handle missing modules gracefully (empty modules dir on first run is fine)

    Create src/core/commands.ts:
    - Export a CommandRegistry class with:
      - register(name, builder, handler): add a command
      - getBuilders(): return all SlashCommandBuilder instances (for deploy-commands.ts)
      - handleInteraction(interaction, ctx): route ChatInputCommandInteraction to correct handler
    - Wire up the interactionCreate event: on interaction, look up command name in registry, call handler
    - All command handlers receive the full ModuleContext
    - Handle unknown commands gracefully (ephemeral "Unknown command" reply)

    Create src/core/events.ts:
    - Simple typed event bus for cross-module communication
    - EventBus class with on(event, handler), emit(event, ...args)
    - Predefined event types: 'memberSetupComplete', 'accountLinked', 'profileUpdated'
    - This is for internal module-to-module communication, not Discord events

    Create src/index.ts (entry point):
    - Import and call config validation first (fail fast)
    - Create Discord client via createClient()
    - Initialize winston logger (console + file transports, JSON format in production)
    - Create PrismaClient (import from db/client.ts -- will be created in Plan 02, for now use a placeholder import that will compile once Plan 02 runs)
    - Build ModuleContext: { client, db, commands, logger }
    - Call loadModules(ctx)
    - On client 'ready' event: log "Bot ready" with guild count, register slash commands with Discord API
    - Call client.login(config.BOT_TOKEN)
    - Handle SIGINT/SIGTERM: disconnect client, close DB, exit cleanly

    IMPORTANT: For the DB import, create a minimal placeholder at src/db/client.ts that exports a null PrismaClient (Plan 02 will replace this with the real implementation). This allows the project to compile and the module system to work even before the database is set up.
