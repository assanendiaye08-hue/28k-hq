    Create src/modules/identity/linking.ts:
    - Export generateLinkCode(db, requesterId: string): Promise<{ code: string, expiresAt: Date }>
      - Generate 6-character uppercase alphanumeric code: crypto.randomBytes(3).toString('hex').toUpperCase()
      - Store LinkCode record: { code, requesterId, expiresAt: now + 5 minutes, used: false }
      - If the requester already has a pending (unused, non-expired) code, delete it first (one active code per account)
      - Return the code and expiry time

    - Export verifyLinkCode(db, verifierId: string, code: string): Promise<{ success: boolean, error?: string, memberId?: string }>
      - MUST use prisma.$transaction for atomicity (prevents race conditions -- see research pitfall #8)
      - Inside transaction:
        1. Look up LinkCode where code matches, not expired, not used
        2. If not found: return { success: false, error: 'invalid_or_expired' }
        3. Mark code as used
        4. Look up the requester's DiscordAccount to find their memberId
        5. If requester has no Member yet: return { success: false, error: 'requester_not_setup' }
        6. Check if verifier already has a DiscordAccount linked to a DIFFERENT member: return { success: false, error: 'already_linked_different' }
        7. Check account cap: if member already has 5 linked accounts, return { success: false, error: 'cap_reached' }
        8. Create DiscordAccount for verifier pointing to the same memberId
        9. Return { success: true, memberId }

    - Export unlinkAccount(db, discordId: string): Promise<{ success: boolean, error?: string }>
      - Look up the DiscordAccount for discordId
      - Check: the member must have at least 2 linked accounts (cannot unlink the last one)
      - If only 1 account: return { success: false, error: 'last_account' }
      - Delete the DiscordAccount record
      - Return { success: true }

    - Export getLinkedAccounts(db, memberId: string): Promise<DiscordAccount[]>
      - Return all DiscordAccounts for a member (used by /profile and /unlink UI)

    Create src/modules/identity/commands.ts:
    - Export linkCommand handler for /link:
      1. deferReply({ flags: MessageFlags.Ephemeral })
      2. Check if the caller has completed /setup (has a DiscordAccount linked to a Member). If not: "You need to run /setup first."
      3. Call generateLinkCode(db, interaction.user.id)
      4. Reply with the code in a prominent embed:
         - "Your link code: **{CODE}**"
         - "Run `/verify {CODE}` from your other Discord account within 5 minutes."
         - Show expiry time
      5. The code is ephemeral (only visible to the caller)

    - Export verifyCommand handler for /verify:
      1. deferReply({ flags: MessageFlags.Ephemeral })
      2. Get the code from the required string option
      3. Call verifyLinkCode(db, interaction.user.id, code.toUpperCase())
      4. On success: "Account linked! This account is now connected to your identity. You share the same profile, XP, and data."
         - Also assign the Member role to the verifying account (so they see all channels)
         - Copy any rank roles from the existing linked accounts
      5. On error: descriptive message per error type
         - 'invalid_or_expired': "That code is invalid or has expired. Generate a new one with /link."
         - 'requester_not_setup': "The account that generated this code hasn't completed /setup yet."
         - 'already_linked_different': "This account is already linked to a different member. Use /unlink first."
         - 'cap_reached': "You've reached the maximum of 5 linked accounts."

    - Export unlinkCommand handler for /unlink:
      1. deferReply({ flags: MessageFlags.Ephemeral })
      2. Get all linked accounts for this member
      3. If only 1 account: "You can't unlink your only account."
      4. If 2+ accounts: show a select menu listing all linked accounts (show Discord username for each)
      5. On selection: call unlinkAccount(db, selectedDiscordId)
      6. Confirm: "Account unlinked. That Discord account is no longer connected to your identity."
