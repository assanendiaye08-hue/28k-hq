    Create the onboarding module. This handles the /setup command, runs a conversational DM flow, creates the member's private space, and unlocks the server.

    Create src/modules/onboarding/welcome.ts:
    - Export sendWelcomeMessage(channel: TextChannel) function
    - Build an embed with the hustler manifesto for #welcome:
      - Title: something bold but not cringe (e.g., "Welcome to the Grind")
      - Description: Short manifesto -- 3-4 lines max. Real talk energy. Not corporate motivational poster.
        Example tone: "This server isn't about watching from the sidelines. You're here because you're building something. Whether it's a business, a skill, or a side hustle -- this is where you show up, put in work, and level up."
      - Fields: Brief overview of how the server works (XP, ranks, private space, AI assistant)
      - Footer: "Run /setup to get started"
    - Use the brand colors from shared/embeds.ts
    - This is sent once when the bot sets up #welcome, and also shown to new members via embed reference

    Create src/modules/onboarding/setup-flow.ts:
    - Export runSetupFlow(member: GuildMember, db: PrismaClient, logger: Logger) function
    - This is the CORE of onboarding. It runs a sequential DM conversation using awaitMessages.

    Flow steps:
    1. Open DM channel (wrap in try-catch -- if DMs disabled, return error to caller)
    2. Send intro message: "Hey! Let's get you set up. I'm going to ask you a few questions so I can tailor the server to you. Just answer naturally -- no wrong answers."
    3. Ask questions one at a time (use awaitMessages with 5-min timeout per question):
       - "What's your current hustle? What are you working on right now?"
       - "What areas are you most into? (coding, content creation, ecom, freelancing, design, whatever)"
       - "What's the main thing you're trying to achieve in the next 3 months?"
       - "What do you want to learn or get better at?"
       - "How do you like to work? Solo grinder? Need accountability? Love competition? Describe your style."
    4. After each answer, send a brief acknowledgment ("Got it.", "Nice.", "Solid.", etc. -- vary them, keep it natural)
    5. After all questions, ask about private space preference:
       - "Last thing -- where do you want your private space? This is where I'll send you briefs, track your stuff, and where you can talk to the AI assistant."
       - Option 1: "Right here in DMs (fully private, just you and me)"
       - Option 2: "A private channel in the server (more integrated, still only visible to you)"
       - "Reply 1 or 2"
    6. Collect the space preference
    7. Return a structured result: { answers: Record<string, string>, spaceType: 'DM' | 'CHANNEL', displayName: string (use their Discord display name) }

    Error handling:
    - If DMs are closed: return { error: 'dms_closed' }
    - If timeout on any question: send "No worries, you can run /setup again whenever you're ready" and return { error: 'timeout' }
    - If member leaves server during setup: handle gracefully

    Create src/modules/onboarding/channel-setup.ts:
    - Export createPrivateChannel(guild: Guild, member: GuildMember, botId: string): Promise<TextChannel>
    - Create a text channel named `space-{username}` under the PRIVATE SPACES category
    - Permission overwrites:
      - @everyone: DENY ViewChannel
      - member: ALLOW ViewChannel, SendMessages, ReadMessageHistory, AttachFiles, EmbedLinks
      - bot: ALLOW ViewChannel, SendMessages, ReadMessageHistory, ManageMessages
    - Send an initial message in the channel: "This is your private space. Your goals, progress, and AI conversations live here. Only you and I can see this channel."
    - Return the created channel

    Create src/modules/onboarding/commands.ts:
    - Export the /setup command handler
    - Flow:
      1. Immediately deferReply({ flags: MessageFlags.Ephemeral }) -- CRITICAL: must respond within 3 seconds, AI + DB calls take longer
      2. Check if member already has "Member" role (already set up) -- if yes, reply "You're already set up!"
      3. Call runSetupFlow(member, db, logger)
      4. If error 'dms_closed': editReply with instructions to enable DMs
      5. If error 'timeout': editReply "Setup timed out. Run /setup again when you're ready."
      6. On success:
         a. Create Member record in DB with generated encryptionSalt
         b. Create DiscordAccount record linking their Discord ID to the Member
         c. Derive member encryption key, generate recovery key
         d. Store raw answers as encrypted JSON in MemberProfile.rawAnswers
         e. Store the answers temporarily for AI processing (Plan 04 will handle AI tag extraction -- for now, store rawAnswers and leave structured tag fields empty)
         f. Create PrivateSpace record (DM or CHANNEL type)
         g. If CHANNEL type: call createPrivateChannel()
         h. Send recovery key to member via DM: "Here's your personal recovery key. Save it somewhere safe -- it can decrypt your data independently if needed." + code block with DHKEY-xxx
         i. Assign "Member" role to the member (this unlocks all gated channels)
         j. editReply "You're all set! Check out the server -- everything is unlocked now." (or point them to their private channel if they chose CHANNEL)
         k. Emit 'memberSetupComplete' event via EventBus

    Create src/modules/onboarding/index.ts:
    - Standard module registration pattern
    - Register /setup command
    - Listen for guildMemberAdd to send a DM or @mention in #welcome prompting them to run /setup
