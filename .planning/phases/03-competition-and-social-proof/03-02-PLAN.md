    `src/modules/leaderboard/calculator.ts`:
    Three query functions. All accept the Prisma client (use ExtendedPrismaClient from db/client.ts) and optional season date range.

    1. `getXPLeaderboard(db, options?: { seasonStart?: Date, seasonEnd?: Date, limit?: number })`:
       - If seasonStart/End provided: Query XPTransaction grouped by memberId where createdAt within range, sum amounts, order desc, take limit. Join with Member for displayName.
       - If no date range: Query Member.findMany ordered by totalXp desc, take limit.
       - Return: Array of { position, memberId, displayName, value } where value = XP.

    2. `getVoiceLeaderboard(db, options?: { seasonStart?: Date, seasonEnd?: Date, limit?: number })`:
       - Query VoiceSession grouped by memberId where startedAt >= seasonStart and endedAt <= seasonEnd (and endedAt is not null), sum durationMinutes, order desc, take limit.
       - Join with Member for displayName.
       - Return: Array of { position, memberId, displayName, value } where value = total minutes.

    3. `getStreakLeaderboard(db, limit?: number)`:
       - Query Member.findMany ordered by currentStreak desc, take limit.
       - Return: Array of { position, memberId, displayName, value } where value = currentStreak.
       - Note: Streaks are NOT seasonal -- they represent lifetime consistency.

    4. `getMemberPosition(db, memberId, dimension, options?)`:
       - For the /leaderboard command's "your position" feature.
       - Query count of members with higher value in the given dimension.
       - Return position number (1-based).

    `src/modules/leaderboard/renderer.ts`:
    Three embed builder functions. Use BRAND_COLORS from shared/constants.ts.

    1. `buildXPLeaderboardEmbed(entries, seasonLabel?)`:
       - Title: "XP Leaderboard" (or "XP Leaderboard -- Season {N}" if seasonal)
       - Color: BRAND_COLORS.primary (amber/gold)
       - Description: Numbered list with medal emojis for top 3 (gold/silver/bronze). Format: `{medal} **{position}. {displayName}** -- {value.toLocaleString()} XP`
       - Footer: "Updated every 15 minutes" with timestamp

    2. `buildVoiceLeaderboardEmbed(entries, seasonLabel?)`:
       - Title: "Voice Hours Leaderboard" (or with season)
       - Color: BRAND_COLORS.info (blue)
       - Description: Same numbered format. Convert minutes to "Xh Ym" format using date-fns intervalToDuration + formatDuration (or manual formatting for simplicity).
       - Footer: "Updated every 15 minutes" with timestamp

    3. `buildStreakLeaderboardEmbed(entries)`:
       - Title: "Streak Leaderboard"
       - Color: BRAND_COLORS.success (green)
       - Description: Same numbered format. Value shown as "{value} days" with fire emoji for streaks >= 7.
       - Footer: "Updated every 15 minutes" with timestamp

    Helper: `getMedalEmoji(position)`: 1 -> gold medal, 2 -> silver medal, 3 -> bronze medal, else -> position number with period.

    All embeds should be clean and compact. No unnecessary fields or sections. The "gym scoreboard" aesthetic -- glanceable data.

    `src/modules/leaderboard/channel-sync.ts`:
    Manages the 3 persistent messages in #leaderboard channel.

    1. `initLeaderboardChannel(client, db)`:
       - Find #leaderboard channel by name in the guild. If it doesn't exist, create it under "The Hub" category with bot-only send permissions (everyone can read but only bot can send). Deny SendMessages for @everyone, allow for bot.
       - Check BotConfig for stored message IDs (3 keys from constants).
       - For each missing message ID: send a placeholder embed to the channel, store the new message ID in BotConfig via upsert.
       - For each existing message ID: try to fetch the message. If DiscordAPIError (Unknown Message -- message was deleted), recreate and update BotConfig.
       - Return the 3 message objects or IDs for use in refresh.

    2. `refreshLeaderboardMessages(client, db)`:
       - Get active Season (db.season.findFirst({ where: { active: true } })). If no active season, use all-time data for XP and voice.
       - Call all 3 calculator functions with season date range (if active season exists).
       - Build all 3 embeds via renderer functions (pass season label like "Season {number}" if applicable).
       - Fetch stored message IDs from BotConfig.
       - For each: try to fetch the message from the #leaderboard channel and edit with new embed. If message gone (catch error), recreate via initLeaderboardChannel flow and store new ID.
       - Log refresh: "Leaderboard refreshed: {xp_count} XP entries, {voice_count} voice entries, {streak_count} streak entries"

    **Important:** Message edits do NOT generate notifications. This satisfies the user's "no new messages, no notifications" requirement. The channel is for consultation only.

    `src/modules/leaderboard/commands.ts`:
    1. `buildLeaderboardCommand()`: SlashCommandBuilder for /leaderboard with a required string option "type" with choices: { name: 'XP', value: 'xp' }, { name: 'Voice Hours', value: 'voice' }, { name: 'Streaks', value: 'streaks' }. Use addStringOption with setRequired(true) and addChoices.
    2. `handleLeaderboard(interaction, ctx)`:
       - Get the selected type from interaction options.
       - Get active season for date range (if applicable).
       - Call the appropriate calculator function.
       - Additionally call getMemberPosition for the invoking member to show "Your position: #X" if they're not in the top 10.
       - Resolve the invoking member: db.discordAccount.findUnique({ where: { discordId: interaction.user.id } }).
       - Build the appropriate embed.
       - If the member is not in the top 10 entries, append a field to the embed: "Your Position" with value "#X with Y XP/hours/days".
       - Reply with the embed (ephemeral: false -- leaderboard checks are public and social).
