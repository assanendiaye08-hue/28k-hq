    **Voice Tracker Module:**

    `src/modules/voice-tracker/constants.ts`:
    - MIN_SESSION_MINUTES = 5 (minimum session to count)
    - NOTEWORTHY_THRESHOLD_MINUTES = 90 (announce sessions >= 1.5 hours)
    - VOICE_CATEGORY_NAME = 'Lock In' (matches SERVER_CATEGORIES.voice.name)
    - PERIODIC_FLUSH_INTERVAL_MS = 10 * 60 * 1000 (flush active session durations every 10 min as safety net)

    `src/modules/voice-tracker/tracker.ts`:
    - ActiveSession interface: { memberId: string, discordId: string, channelId: string, startedAt: Date, pausedAt?: Date, totalPausedMs: number }
    - In-memory Map<string, ActiveSession> keyed by discordId (Discord user ID)
    - `startSession(discordId, channelId, memberId)`: Create entry in map. Emit voiceSessionStarted event.
    - `endSession(discordId, db, events)`: Calculate duration = (now - startedAt - totalPausedMs). If durationMinutes >= MIN_SESSION_MINUTES: persist VoiceSession to DB (find active season for seasonId), calculate voice XP (Math.floor(durationMinutes / 3) capped at daily remaining from 200 cap -- query today's voice XP transactions), call awardXP with VOICE_SESSION source. Emit voiceSessionEnded. Remove from map. Return { durationMinutes, xpAwarded, noteworthy: durationMinutes >= NOTEWORTHY_THRESHOLD }.
    - `pauseSession(discordId)`: Set pausedAt = now on the active session.
    - `resumeSession(discordId)`: Add (now - pausedAt) to totalPausedMs, clear pausedAt.
    - `updateSessionChannel(discordId, newChannelId)`: Update channelId in the active session (for co-work channel switches).
    - `getActiveSession(discordId)`: Return session or undefined.
    - `reconstructSessions(guild, db)`: On bot ready, scan all voice channels under "Lock In" category. For each member found in a voice channel (not the AFK channel), create a session with startedAt = now. Log reconstruction count.
    - Helper: `isTrackedChannel(channelId, guild)`: Check if the channel is under the "Lock In" category and is not the guild's AFK channel.
    - Helper: `getTodayVoiceXP(db, memberId)`: Query sum of XPTransaction where source = VOICE_SESSION and createdAt >= start of today (UTC). Used for daily cap check.

    `src/modules/voice-tracker/embeds.ts`:
    - `buildSessionSummaryEmbed(displayName, durationMinutes, xpAwarded)`: Encouraging embed for noteworthy sessions. Example: "{durationMinutes} locked in. That's how you do it." Use BRAND_COLORS.success. Include XP earned. Keep it brief -- one-liner energy.

    `src/modules/voice-tracker/index.ts`:
    - Module registration following existing pattern: `{ name: 'voice-tracker', register(ctx) }`
    - In register:
      1. Listen to `client.on('voiceStateUpdate', (oldState, newState) => ...)` directly on the discord.js client (NOT the event bus -- this is a Discord event, not a bot event).
      2. voiceStateUpdate handler logic:
         - Resolve memberId from discordId via db.discordAccount.findUnique. If not found, ignore (not a registered member).
         - Get guild.afkChannelId for AFK detection.
         - JOIN (oldChannel null, newChannel not null): If isTrackedChannel, startSession.
         - LEAVE (oldChannel not null, newChannel null): If has active session, endSession. If noteworthy, deliver summary embed to private space via deliverToPrivateSpace.
         - SWITCH: If newChannel is AFK channel -> pauseSession. If oldChannel was AFK channel -> resumeSession. If both are tracked channels -> updateSessionChannel. If oldChannel was tracked but newChannel is not tracked (and not AFK) -> endSession.
         - SERVER-DEAFEN: If newState.serverDeaf && !oldState.serverDeaf -> pauseSession. If !newState.serverDeaf && oldState.serverDeaf -> resumeSession.
      3. On `client.once('ready')`: Call reconstructSessions to recover from restarts.
    - Self-deafened: continue tracking (per research discretion recommendation -- deep focus).
    - Self-muted: continue tracking (per user decision).

    **Wins/Lessons Module:**

    `src/modules/wins-lessons/constants.ts`:
    - WINS_CHANNEL_NAME = 'wins' (matches SERVER_CATEGORIES)
    - LESSONS_CHANNEL_NAME = 'lessons'
    - WIN_EMOJI = '\uD83D\uDCAA' (muscle)
    - LESSON_EMOJI = '\uD83E\uDDE0' (brain)

    `src/modules/wins-lessons/handler.ts`:
    - In-memory cooldown Map<string, Map<string, number>> keyed by memberId -> type ('win'|'lesson') -> last award timestamp
    - `isOnCooldown(memberId, type)`: Check if last award was within winsLessonsCooldownMs (2 hours). Return boolean.
    - `setCooldown(memberId, type)`: Record current timestamp.
    - `handleWinsLessonsMessage(message, db, events, client)`:
      1. Ignore bot messages (message.author.bot).
      2. Look up channel: find #wins and #lessons channels by name in the guild. Compare message.channelId.
      3. If not in #wins or #lessons, return.
      4. Resolve member: db.discordAccount.findUnique({ where: { discordId: message.author.id } }). If not found, return.
      5. Determine type (win/lesson) and check cooldown. If on cooldown, still react (acknowledgement) but don't award XP.
      6. React with appropriate emoji (WIN_EMOJI or LESSON_EMOJI). No reply -- just the reaction.
      7. If not on cooldown: award XP via awardXP(db, memberId, amount, source, description). Set cooldown. Emit winPosted/lessonPosted event.
      8. If levelUp detected from awardXP result, emit levelUp event (existing pattern handles role sync).

    `src/modules/wins-lessons/index.ts`:
    - Module registration: `{ name: 'wins-lessons', register(ctx) }`
    - In register: Listen to `client.on('messageCreate', (message) => handleWinsLessonsMessage(message, db, events, client))`.
