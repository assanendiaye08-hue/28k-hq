<objective>
Build the auto-content feed system that fetches from RSS, YouTube, and Reddit, filters through DeepSeek V3.2 AI for quality, and posts 2-4 curated items per day into #auto-feed with member reaction feedback.

Purpose: Gives members a steady stream of actionable, relevant content without anyone needing to manually curate. The AI filter ensures quality over quantity -- only content that provides concrete techniques, tools, or resources gets posted. Member upvote/downvote reactions train the filter over time by building per-source and per-category approval rates.

Output: Working auto-feed module that runs on a cron schedule, fetches from configured sources, AI-filters for relevance, posts rich embeds to #auto-feed, and tracks member feedback.
</objective>

    - Export `fetchReddit(subreddit: string, sort: string = 'hot', limit: number = 10): Promise<FeedItem[]>` -- uses native fetch() with REDDIT_USER_AGENT header to `https://www.reddit.com/r/${subreddit}/${sort}.json?limit=${limit}`. Parse response JSON, map `data.data.children` to FeedItem array with source: 'reddit'. Filter out stickied posts and posts with score < 10 (basic pre-filter). Wrap in try/catch returning empty array on failure.
    - Export `fetchAllSources(db: ExtendedPrismaClient): Promise<FeedItem[]>` -- queries all active FeedSource records from DB, dispatches to the appropriate fetcher based on FeedType (RSS -> fetchRSS, YOUTUBE -> fetchYouTube, REDDIT -> fetchReddit). Concatenates all results. Add a 500ms delay between Reddit fetches to respect rate limits (use a simple `await new Promise(r => setTimeout(r, 500))` between each Reddit source).

      1. For each item, query FeedPost to check if already posted (sourceId + externalId combo -- but since we don't have sourceId here yet, deduplicate by checking if any FeedPost with matching `link` exists). If already posted, skip.

      3. System prompt: "You are a content curator for a productivity Discord server. Members are diverse: FAANG engineers, small business owners, students, ecom operators, content creators, designers. Classify each content item. Return a JSON classification."

      5. User prompt: "Classify this content:\nTitle: {title}\nSource: {sourceName}\nSummary: {content sliced to 500 chars}"
      6. Parse response into FilterResult. Only keep items where `keep === true && relevanceScore >= MIN_RELEVANCE_SCORE`.
      7. Batch AI calls: process items sequentially (not parallel) to stay within rate limits. Use try/catch per item -- if AI call fails for one item, skip that item and continue.
      8. If ALL AI calls fail (OpenRouter down), return empty array. Never post unfiltered content.
      9. Sort kept items by relevanceScore descending. Return top MAX_POSTS_PER_CYCLE items.

    **Create src/modules/auto-feed/poster.ts:**
    - Export `postFeedItems(client: Client, db: ExtendedPrismaClient, items: Array<FeedItem & { filter: FilterResult }>, guildId: string): Promise<number>`:
      1. Find the #auto-feed channel by name within the RESOURCES category (find category named 'RESOURCES', then find text channel named 'auto-feed' with that category as parent). If channel not found, log warning and return 0.
      2. For each item, check deduplication: query FeedPost where link = item.link. If exists, skip.
      3. Build a rich embed for each item:
         - Color: source-based (RSS: 0x3b82f6 blue, YouTube: 0xff0000 red, Reddit: 0xff4500 orange-red)
         - Title: item.title (truncated to 256 chars)
         - URL: item.link
         - Description: item.content (truncated to 200 chars) + `\n\nCategory: ${filter.category} | Score: ${filter.relevanceScore}/100`
         - Footer: `via ${item.sourceName}` + timestamp of item.publishedAt
         - Author: item.author if present
      4. Send embed to channel. After successful send, add UPVOTE_EMOJI and DOWNVOTE_EMOJI reactions to the message.
      5. Create FeedPost record in DB: find the FeedSource by matching the sourceName, set externalId to item.link (URL as unique identifier), title, link, category from filter, relevanceScore from filter, messageId from the sent Discord message.
      6. Return count of successfully posted items.

    **Create src/modules/auto-feed/feedback.ts:**
    - Export `collectFeedback(db: ExtendedPrismaClient, client: Client, guildId: string): Promise<void>`:
      1. Query all FeedPost records from the last 7 days that have a messageId.
      2. For each post, fetch the Discord message by ID from the #auto-feed channel.
      3. Count thumbs-up and thumbs-down reactions (subtract 1 from each for the bot's own reactions).
      4. Update the FeedPost record with upvotes and downvotes counts.
      5. Wrap each message fetch in try/catch -- message may have been deleted.
    - This runs less frequently (once daily) to update feedback stats that the AI filter uses.

    **Create src/modules/auto-feed/index.ts:**

    - In register:
      1. On client 'ready' event, set up cron jobs using node-cron schedule() for each FEED_CRON_SCHEDULES entry. Each cron job runs the full pipeline: `fetchAllSources(db) -> filterItems(items, db) -> postFeedItems(client, db, filtered, guildId)`. Get guildId from the first guild the client is in (client.guilds.cache.first().id).
      2. Set up one additional daily cron job at '0 3 * * *' (3 AM UTC) for `collectFeedback(db, client, guildId)`.
      3. Log startup message: "Auto-feed module loaded: {N} cron schedules registered".
