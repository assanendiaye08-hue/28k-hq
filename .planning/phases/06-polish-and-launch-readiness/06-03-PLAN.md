    **Create src/modules/hardening/member-lifecycle.ts:**

    Export `handleMemberRemove(member: GuildMember, db: ExtendedPrismaClient, logger: Logger): Promise<void>`:
    1. Look up DiscordAccount by discordId = member.id.
    2. If found, log: "Member left: {displayName} (memberId: {memberId}). Data preserved -- use /deletedata to remove."
    3. Do NOT delete any data. The user explicitly left the server but may rejoin. Data deletion is only via /deletedata.
    4. Post a brief note to #bot-log: "{displayName} left the server. Profile data preserved."

    Export `handleMemberAdd(member: GuildMember, client: Client, db: ExtendedPrismaClient, logger: Logger): Promise<void>`:
    1. Look up DiscordAccount by discordId = member.id.
    2. If NOT found: this is a genuinely new member. Do nothing (the existing onboarding module handles guildMemberAdd with a welcome message and /setup prompt).
    3. If FOUND: this is a rejoin. The member has an existing profile.
       a. Try to DM the member: "Welcome back! Your profile is still here. Want to pick up where you left off, or start fresh?\n\nReply **restore** to keep your existing profile, or **fresh** to wipe everything and start over with /setup."
       b. Use awaitMessages with a 5-minute timeout (reuse the SETUP_TIMEOUT_MS constant pattern).
       c. If they reply "fresh" or "start fresh" (case-insensitive): Call the deleter function from data-privacy module (import `deleteMemberData`) to wipe their data, then tell them "Done! Run /setup to create a new profile."
       d. If they reply "restore" or anything else (default to restore): Reassign the Member role so they regain channel access. Reply "Welcome back! Everything is just how you left it." Also reassign any interest tag roles they previously had (query MemberProfile for interests, find matching InterestTag roles, add them).
       e. If timeout: Default to restore (non-destructive). Assign Member role silently. Post to #bot-log: "{displayName} rejoined, timed out on choice, auto-restored."
    4. Post rejoin event to #bot-log: "{displayName} rejoined. Choice: {restore|fresh|timeout-restore}."

      3. Listen for 'guildMemberAdd' event: call `handleMemberAdd(member, ctx.client, db, ctx.logger)`.
