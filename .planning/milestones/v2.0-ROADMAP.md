# Roadmap: 28K HQ v2.0 Desktop Companion App

## Overview

This milestone transforms the Discord bot into a three-application platform by extracting shared code into a monorepo, building a Fastify REST API, and creating a Tauri v2 desktop companion app. The desktop app puts a productivity timer in the menu bar, goals in a visual hierarchy, and a daily dashboard one click away. Phases progress from infrastructure (monorepo) through data layer (API + auth) to presentation (desktop shell, timer modes, goals editing) and finally cross-platform integration (bot + desktop sync). The monorepo restructure is first because everything depends on it; the cross-platform sync is last because it is the hardest integration point.

## Phase Details

### Phase 14: Monorepo Restructure
**Goal**: Codebase is organized as a Turborepo + pnpm monorepo with shared packages, and the existing bot runs identically from its new location
**Depends on**: Nothing (first phase of v2.0)
**Requirements**: INFRA-01, INFRA-02, INFRA-03, INFRA-04
**Success Criteria** (what must be TRUE):
  1. Running `pnpm dev --filter bot` from the monorepo root starts the Discord bot and it connects, loads all modules, and responds to commands identically to the pre-migration state
  2. `packages/db` exports the Prisma client and schema, and both `apps/bot` and a test import in `apps/api` can import and use it without TypeScript errors (TS2742 resolved)
  3. `packages/shared` exports XP engine functions, rank progression constants, timer constants, and brand colors, and `apps/bot` imports them instead of local copies
  4. `pnpm install` from root resolves all dependencies with strict mode (no phantom dependencies)

### Phase 15: REST API + Authentication
**Goal**: A Fastify REST API server is running on the VPS with Discord OAuth login and all endpoints the desktop app needs for timer, goals, dashboard, and quotes
**Depends on**: Phase 14
**Requirements**: API-01, API-02, API-03, API-04, API-05, API-06, AUTH-01, AUTH-02, AUTH-03
**Success Criteria** (what must be TRUE):
  1. A user can complete Discord OAuth login (PKCE flow) and receive a JWT access token + refresh token pair
  2. Authenticated requests to timer endpoints can create, pause, resume, stop, and query timer sessions, with XP awarded on completion using shared logic from packages/shared
  3. Authenticated requests to goals endpoints return the full goal hierarchy with progress, and can create goals, update progress, and mark goals complete
  4. Authenticated requests to dashboard endpoint return today's priorities, weekly goals, streak, rank, XP, and a daily rotating quote
  5. Unauthenticated requests are rejected with 401, and expired tokens can be refreshed

### Phase 16: Desktop Shell + Dashboard + Goals View
**Goal**: Users can install and launch the Tauri desktop app, log in with Discord, and see their dashboard (priorities, streak, rank, quote) and goal hierarchy in a dark-themed UI with the gold ouroboros in the system tray
**Depends on**: Phase 15
**Requirements**: APP-01, APP-02, APP-03, APP-04, APP-05, AUTH-04, DASH-01, DASH-02, DASH-03, DASH-04, DASH-05, GOAL-01, GOAL-02
**Success Criteria** (what must be TRUE):
  1. User can download and launch the app on macOS and Windows, see the gold ouroboros icon in the system tray / menu bar, and click it to open a popover showing current state
  2. User can log in via Discord OAuth, and their tokens are stored in OS keychain (Tauri secure storage) so they remain logged in across app restarts
  3. User can open the main window and see a dashboard with today's active goals, this week's goals, their current streak (fire visual if active), rank with XP progress bar, and a daily operator quote
  4. User can navigate to a goals view showing the full hierarchy (yearly to weekly) as a nested list with progress bars for measurable goals and completion status for freetext goals
  5. The entire UI uses the dark theme with gold accents matching the brand identity

### Phase 17: Pomodoro Timer
**Goal**: Users can run a full pomodoro timer from the desktop app with configurable intervals, menu bar countdown, popover controls, alarm transitions, and session data persisted to the API
**Depends on**: Phase 16
**Requirements**: TMR-01, TMR-02, TMR-03, TMR-04, TMR-05, TMR-06, TMR-07, TMR-08, TMR-09, TMR-10, TMR-11, TMR-12
**Success Criteria** (what must be TRUE):
  1. User can configure work duration, break duration, session count, and long break interval on a setup screen, enter a focus description, and start a pomodoro timer
  2. Running timer countdown displays in the menu bar (macOS: tray title text, Windows: mini-window fallback) and clicking it opens a popover with pause/stop/change-focus controls
  3. When a work interval ends, the app comes to foreground with an alarm sound; user can auto-start break or manually trigger it; when break ends, the same foreground + alarm behavior occurs
  4. Completed session data syncs to the API (XP awarded, streak tracked), and timer state persists across app restarts so the user can resume where they left off

### Phase 18: Flowmodoro + Goals Editing
**Goal**: Users can run a flowmodoro (count-up) timer with auto-calculated breaks, and can create, update, and complete goals from the desktop app
**Depends on**: Phase 17
**Requirements**: FLW-01, FLW-02, FLW-03, FLW-04, FLW-05, FLW-06, GOAL-03, GOAL-04, GOAL-05
**Success Criteria** (what must be TRUE):
  1. User can start a running timer that counts up with no fixed duration, configure the work-to-break ratio, and when they pause or stop, the break duration is auto-calculated from the ratio
  2. Running flowmodoro timer displays elapsed time in the menu bar and the popover provides the same controls as pomodoro (pause/stop/change focus)
  3. Flowmodoro session data syncs to the API on completion (same XP/streak path as pomodoro)
  4. User can create a new goal from the desktop app (title, deadline, type, parent), update progress on measurable goals, and mark goals complete (XP awarded via API)

### Phase 19: Cross-Platform Sync + Polish
**Goal**: Timer and goal state syncs between the desktop app and Discord bot in real-time, with platform enforcement (one timer at a time) and distribution polish (auto-updater, autostart)
**Depends on**: Phase 18
**Requirements**: SYNC-01, SYNC-02, SYNC-03, SYNC-04, GOAL-06, DASH-06, APP-06, APP-07
**Success Criteria** (what must be TRUE):
  1. Timer started in the desktop app is visible to the Discord bot (bot can check status and report it), and when a desktop timer completes, the bot sends a completion DM with XP summary
  2. Only one active timer per member is enforced across both platforms -- starting a desktop timer while a bot timer is running (or vice versa) is blocked with a clear message
  3. Goals created or completed on either platform are immediately visible on the other (bot creates goal, app shows it; app completes goal, bot reflects it)
  4. Dashboard updates in real-time when timer completes or goals change without requiring manual refresh
  5. Auto-updater via GitHub Releases and autostart on login are available as opt-in settings (both unchecked by default on first launch)
