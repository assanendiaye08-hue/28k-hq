# Requirements: 28K HQ v2.0 Desktop Companion App

**Defined:** 2026-03-21
**Core Value:** When a member opens Discord, the environment pulls them into productive action. The desktop app extends this to their entire workstation — the timer, goals, and dashboard are always one click away.

## v2.0 Requirements

### Infrastructure

- [ ] **INFRA-01**: Codebase is restructured into a Turborepo + pnpm monorepo with apps/bot, apps/desktop, apps/api, packages/db, packages/shared
- [ ] **INFRA-02**: Existing Discord bot builds and runs identically after monorepo migration (zero functional regression)
- [ ] **INFRA-03**: Prisma schema and client are extracted into packages/db and shared by bot and API
- [ ] **INFRA-04**: Shared business logic (XP engine, rank progression, timer constants, brand colors) is extracted into packages/shared

### API

- [ ] **API-01**: Fastify REST API server runs on VPS alongside the Discord bot
- [ ] **API-02**: API exposes timer CRUD endpoints (start, pause, resume, stop, status)
- [ ] **API-03**: API exposes goals endpoints (list hierarchy, create, update progress, complete)
- [ ] **API-04**: API exposes dashboard endpoints (today's priorities, streak, rank, XP, weekly goals)
- [ ] **API-05**: API exposes daily quote endpoint (rotating operator quotes from curated pool)
- [ ] **API-06**: API uses shared XP award logic from packages/shared for timer session completion

### Authentication

- [ ] **AUTH-01**: User can log in to desktop app via Discord OAuth 2.0 (PKCE flow)
- [ ] **AUTH-02**: OAuth maps Discord account to existing Member record in database
- [ ] **AUTH-03**: JWT access tokens (15 min) + refresh tokens (30 days) for API authentication
- [ ] **AUTH-04**: Tokens stored securely in Tauri secure storage (OS keychain)

### Desktop App — Shell

- [ ] **APP-01**: Tauri v2 desktop app builds and runs on macOS and Windows
- [ ] **APP-02**: App icon (gold ouroboros) appears in system tray / menu bar
- [ ] **APP-03**: Clicking tray icon opens a popover/mini-window with current state
- [ ] **APP-04**: App has a full main window for dashboard, goals, and timer setup
- [ ] **APP-05**: Dark theme with gold accents matching brand identity
- [ ] **APP-06**: Auto-updater via GitHub Releases (opt-in, unchecked by default on first launch)
- [ ] **APP-07**: Autostart on login (opt-in, unchecked by default on first launch)

### Timer — Pomodoro

- [ ] **TMR-01**: User can configure pomodoro work duration (1-180 min, default 25)
- [ ] **TMR-02**: User can configure break duration (1-60 min, default 5)
- [ ] **TMR-03**: User can configure number of sessions to chain (1-12, or unlimited)
- [ ] **TMR-04**: User can configure long break duration and interval (e.g., 15 min every 4 sessions)
- [ ] **TMR-05**: User must describe what they're working on before starting (focus field required)
- [ ] **TMR-06**: Timer countdown displays in menu bar while running
- [ ] **TMR-07**: Clicking menu bar timer opens popover with pause/stop/change focus controls
- [ ] **TMR-08**: App comes to foreground with alarm sound when work interval ends
- [ ] **TMR-09**: User can choose auto-start break or manual button to start break
- [ ] **TMR-10**: App comes to foreground with alarm sound when break ends
- [ ] **TMR-11**: Session data syncs to API on completion (XP awarded, streak tracked by bot)
- [ ] **TMR-12**: Timer state persists across app restarts (resume where left off)

### Timer — Flowmodoro

- [ ] **FLW-01**: User can start a running timer that counts up (no fixed duration)
- [ ] **FLW-02**: User can configure work-to-break ratio (default 5:1)
- [ ] **FLW-03**: When user pauses/stops work, break duration is auto-calculated from ratio
- [ ] **FLW-04**: Running timer displays in menu bar (elapsed time, not countdown)
- [ ] **FLW-05**: Same popover controls as pomodoro (pause/stop/change focus)
- [ ] **FLW-06**: Session data syncs to API on completion

### Goals

- [ ] **GOAL-01**: User can view goal hierarchy as a nested list (yearly → quarterly → monthly → weekly)
- [ ] **GOAL-02**: Goals show progress bars (measurable goals: current/target, freetext: complete/not)
- [ ] **GOAL-03**: User can create a new goal from the desktop app (title, deadline, type, parent)
- [ ] **GOAL-04**: User can update progress on a measurable goal
- [ ] **GOAL-05**: User can mark a goal as complete (triggers XP award via API)
- [ ] **GOAL-06**: Goal changes sync bidirectionally (bot creates goal → app sees it, app creates goal → bot sees it)

### Dashboard

- [ ] **DASH-01**: Dashboard shows today's active goals and priorities
- [ ] **DASH-02**: Dashboard shows this week's goals
- [ ] **DASH-03**: Dashboard shows current streak (with fire visual if active)
- [ ] **DASH-04**: Dashboard shows current rank and XP progress to next rank
- [ ] **DASH-05**: Dashboard shows a daily rotating operator quote from curated pool
- [ ] **DASH-06**: Dashboard updates in real-time when timer completes or goals change

### Cross-Platform Sync

- [ ] **SYNC-01**: Timer started in desktop app is visible to Discord bot (bot can check status)
- [ ] **SYNC-02**: Timer completed in desktop app triggers bot to send completion DM with XP
- [ ] **SYNC-03**: One active timer per member enforced across both platforms (can't run bot + desktop timer simultaneously)
- [ ] **SYNC-04**: Goals created/completed on either platform are immediately visible on the other

## v3.0 Requirements (Future)

### Mobile
- **MOB-01**: iOS companion app with same API
- **MOB-02**: Web companion app

### Activity Tracking
- **ACT-01**: Desktop app tracks active window/tab for productivity scoring
- **ACT-02**: Productivity score influences XP multiplier

### Polish
- **POL-01**: Ambient sounds during work sessions (lo-fi, white noise)
- **POL-02**: XP award animations in desktop app
- **POL-03**: Notification center integration for reminders

## Out of Scope

| Feature | Reason |
|---------|--------|
| App/website blocking | Needs elevated OS permissions, not core value |
| Built-in task manager | Duplicates goals system |
| Theme customization | 10-25 users, brand is dark+gold |
| Calendar integration | Not needed for MVP |
| Detailed analytics | Monthly recaps + Jarvis already handle this |
| Linux support | Not enough users to justify testing |
